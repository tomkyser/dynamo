Main dispatch based on `process.argv[2]`:

1. `'start'` -> handleStart():
   - Discover project root via discoverRoot(process.cwd()).
   - Check isDaemonRunning(). If running: print "Dynamo already running (PID {pid}, port {port})", exit 0.
   - Call spawnDaemon(projectRoot, { port }).
   - Call waitForHealth(port, 5000).
   - If health ok: print "Dynamo running (PID {pid}, port {port})", exit 0.
   - If timeout: print error "Daemon failed to start. Check .dynamo/dynamo.log", exit 1.

2. `'stop'` -> handleStop():
   - Read daemon file. If not running: print "Dynamo is not running.", exit 0.
   - POST /shutdown to daemon.
   - Wait up to 5s for daemon PID to die (poll isDaemonRunning every 200ms).
   - Print "Dynamo stopped.", exit 0.

3. `'status'` -> handleStatus():
   - Read daemon file. If not running: print "Dynamo is not running.", exit 0.
   - GET /health. Print formatted output: status, PID, port, uptime, modules list.

4. `'hook'` -> handleHook():
   - The 7-state off-ramp decision tree (per D-04):
     a. Dev bypass: `if (process.env.DYNAMO_DEV_BYPASS === '1') { stdout('{}'); exit(0); }` (state 7)
     b. Read stdin for payload (same Bun.stdin.stream pattern as current).
     c. Discover project root.
     d. Check isDaemonRunning():
        - Not running (no PID): stdout('{}'), exit(0) (state 1 -- silent)
        - Stale PID: stderr warning + cleanup, exit(1) (state 2 -- loud)
        - Running: continue
     e. Read .dynamo/active-triad.json for triad context (if exists).
     f. Build env object: `{ SESSION_IDENTITY: process.env.SESSION_IDENTITY || triad?.faceSessionIdentity, TRIPLET_ID: process.env.TRIPLET_ID || triad?.triadId }`.
     g. POST /hook to daemon with `{ type: process.argv[3], payload, env }`.
     h. If response not ok: stderr error, exit(1) (state 6).
     i. stdout response body, exit(0) (state 5).

5. `'reverie'` + `process.argv[3] === 'kill'` -> Direct kill (no daemon needed):
   - Require and execute the existing kill handler from modules/reverie/components/cli/kill.cjs.
   - This bypasses daemon entirely -- nuclear pre-bootstrap option per D-14.

6. Default (any other command) -> handleCli():
   - Read daemon file. If not running: stderr "Dynamo is not running. Start with: bun bin/dynamo.cjs start", exit(1).
   - POST /cli to daemon with `{ command: process.argv[2], args: process.argv.slice(3), flags: parseSimpleFlags(process.argv) }`.
   - stdout response.output, exit(response.exitCode).

Helper: `parseSimpleFlags(argv)` - extract --json, --raw, --help flags from argv into an object.

Helper: `readTriadFile(projectRoot)` - Read `.dynamo/active-triad.json`, return parsed JSON or null.

NO bootstrap() call anywhere. NO service imports. Only daemon-lifecycle utilities and fetch().

```javascript
async dispatchHook(type, payload, env) {
  if (!_hookRegistry) return {};

  const listeners = _hookRegistry.getListeners(type);
  if (!listeners || listeners.length === 0) return {};

  const enrichedPayload = { ...payload, env };
  let lastResponse = {};

  for (const listener of listeners) {
    try {
      const result = await listener.handler(enrichedPayload);
      if (result && typeof result === 'object' && Object.keys(result).length > 0) {
        lastResponse = result;
      }
    } catch (err) {
      throw new Error(`Hook handler error (${type}, ${listener.serviceName}): ${err.message}`);
    }
  }

  return lastResponse;
}
```
