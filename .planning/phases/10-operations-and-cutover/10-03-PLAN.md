    Create `dynamo/lib/switchboard/sync.cjs` with module identity block `// Dynamo > Switchboard > sync.cjs`.

    Require: `path`, `fs`, `os`. Require core.cjs for `{ DYNAMO_DIR, output, error }`.

    Define constants:
    ```javascript
    const REPO_DIR = path.join(__dirname, '..', '..');  // dynamo/ in repo
    const LIVE_DIR = path.join(os.homedir(), '.claude', 'dynamo');
    const SYNC_STAMP = path.join(REPO_DIR, '.last-sync');
    const SYNC_EXCLUDES = [
      '.env', '.env.example', '.venv', '__pycache__', 'sessions.json',
      'hook-errors.log', '.DS_Store', '.last-sync', 'node_modules',
      'config.json', 'tests'   // tests stay in repo only
    ];
    const GLOB_EXCLUDES = ['*.pyc'];  // pattern-based excludes
    ```

    Implement helper functions:

    **walkDir(dir, excludes, globExcludes, base):**
    - `base = base || dir`
    - `const entries = fs.readdirSync(dir, { withFileTypes: true })`
    - For each entry: skip if name is in excludes array, skip if matches any glob pattern (e.g., `*.pyc` -> endsWith('.pyc'))
    - If directory: recurse. If file: `const stat = fs.statSync(fullPath)`, record `{ mtime: stat.mtimeMs, size: stat.size }` keyed by `path.relative(base, fullPath)`.
    - Return the files map.

    **diffTrees(srcFiles, dstFiles):**
    - `toCopy`: files in src that are missing in dst, or exist in both but src is newer (mtime > dst mtime) or different size
    - `toDelete`: files in dst that don't exist in src
    - Return `{ toCopy, toDelete }`.

    **detectConflicts(srcFiles, dstFiles):**
    - For each file that differs in src->dst direction: check if it also differs in dst->src direction
    - A conflict = file exists in both, both have different content (different size or different mtime from a shared baseline)
    - Since we don't have a baseline, use size difference as the conflict indicator: if sizes differ from both sides' perspective, it's a conflict
    - Simpler approach per research pitfall 3: compare by size first. If sizes equal, compare file content byte-by-byte using `Buffer.compare(fs.readFileSync(srcPath), fs.readFileSync(dstPath))`. Return list of conflicting relative paths.

    **copyFiles(srcDir, dstDir, relativePaths):**
    - For each relative path: ensure parent directory exists via `fs.mkdirSync(path.dirname(dstPath), { recursive: true })`, then `fs.copyFileSync(srcPath, dstPath)`.

    **deleteFiles(dstDir, relativePaths):**
    - For each relative path: `fs.unlinkSync(dstPath)` if it exists.

    Implement `async function run(args = [], pretty = false)`:
    1. Parse args: extract direction (args[0]: 'live-to-repo', 'repo-to-live', 'status'), `--dry-run`, `--force`, `--pretty`.
    2. If no direction or invalid: `error('Usage: dynamo sync <live-to-repo|repo-to-live|status> [--dry-run] [--force]')`.
    3. Walk both trees: `const repoFiles = walkDir(REPO_DIR, SYNC_EXCLUDES, GLOB_EXCLUDES)` and `const liveFiles = walkDir(LIVE_DIR, SYNC_EXCLUDES, GLOB_EXCLUDES)`.
    4. For `status`: diff both directions, output `{ command: 'sync', subcommand: 'status', repo_to_live: diffTrees(repoFiles, liveFiles), live_to_repo: diffTrees(liveFiles, repoFiles), last_sync: lastSyncTimestamp }`.
    5. For `live-to-repo` or `repo-to-live`: determine src/dst dirs and file maps. Check conflicts unless `--force`. If conflicts found, output them and exit with `error()`. If `--dry-run`, output what would change. Otherwise, execute copyFiles and deleteFiles, then write ISO timestamp to SYNC_STAMP.
    6. Build result: `{ command: 'sync', subcommand, direction, files_copied: N, files_deleted: N, dry_run: bool, conflicts: [] }`.
    7. If pretty: format to stderr. Always: `output(result)`.

    Export: `module.exports = { run, walkDir, diffTrees }` (walkDir and diffTrees exported for testing).

    - Test walkDir: correctly walks, respects excludes, skips .pyc files
    - Test diffTrees: identifies toCopy (newer/missing) and toDelete (extra files)
    - Test conflict detection: when both sides have a file with different size, it's a conflict
    - Test that excluded files/dirs are not included in walk results
    - Test copyFiles creates parent directories as needed
    - Use `os.tmpdir()` for all test directories -- never touch real ~/.claude/

    Create `dynamo/lib/switchboard/stack.cjs` with module identity block `// Dynamo > Switchboard > stack.cjs`.

    Require: `path`, `os`, `{ execSync }` from `child_process`. Require core.cjs for `{ output, error, fetchWithTimeout, loadConfig }`. Require pretty.cjs if needed.

    Define constants:
    ```javascript
    const GRAPHITI_DIR = path.join(os.homedir(), '.claude', 'graphiti');
    const COMPOSE_FILE = path.join(GRAPHITI_DIR, 'docker-compose.yml');
    const HEALTH_URL = 'http://localhost:8100/health';
    const MAX_HEALTH_ATTEMPTS = 30;
    const HEALTH_INTERVAL_MS = 2000;
    ```

    Implement `async function start(args = [], pretty = false)`:

    1. Parse args for `--pretty` and `--verbose`.
    2. Check if already running:
       ```javascript
       try {
         const ps = execSync(`docker compose -f "${COMPOSE_FILE}" ps --status running`,
           { encoding: 'utf8', timeout: 10000, stdio: ['pipe', 'pipe', 'pipe'] });
         if (ps.includes('graphiti-neo4j') && ps.includes('graphiti-mcp')) {
           const result = { command: 'start', status: 'already_running', message: 'Graphiti stack is already running' };
           output(result); return;
         }
       } catch (e) { /* not running, proceed */ }
       ```
    3. Start containers:
       ```javascript
       execSync(`docker compose -f "${COMPOSE_FILE}" up -d`,
         { timeout: 60000, stdio: ['pipe', 'pipe', 'pipe'] });
       ```
    4. Health wait loop:
       ```javascript
       for (let i = 0; i < MAX_HEALTH_ATTEMPTS; i++) {
         try {
           const resp = await fetchWithTimeout(HEALTH_URL, {}, 3000);
           if (resp.ok) {
             const result = { command: 'start', status: 'started', message: 'Graphiti stack is ready',
               endpoints: { mcp: 'http://localhost:8100/mcp', neo4j: 'http://localhost:7475' } };
             output(result); return;
           }
         } catch (e) { /* not ready yet */ }
         await new Promise(r => setTimeout(r, HEALTH_INTERVAL_MS));
       }
       ```
    5. If loop exhausts: `output({ command: 'start', status: 'timeout', message: 'Stack started but health check timed out after 60s. Check: docker compose -f "' + COMPOSE_FILE + '" logs graphiti-mcp' })`.

    Implement `async function stop(args = [], pretty = false)`:

    1. Run `execSync(\`docker compose -f "${COMPOSE_FILE}" down\`, { timeout: 30000, stdio: ['pipe', 'pipe', 'pipe'] })`.
    2. Output: `{ command: 'stop', status: 'stopped', message: 'Graphiti stack stopped. Data volumes preserved.', hint: 'To remove data: docker compose -f "' + COMPOSE_FILE + '" down -v' }`.
    3. Catch errors: return FAIL with error message (e.g., Docker not found, compose file missing).

    Export: `module.exports = { start, stop }`.
