<objective>
Wire the Secondary face prompt authority pipeline that verification found missing.

The Context Manager has `receiveSecondaryUpdate()` and `setSecondaryActive()` methods fully implemented and unit-tested, but no production code calls them. At runtime `_secondaryActive` is always `false`, so `compose()` always uses local composition instead of deferring to Secondary's face prompt output.

This plan adds three production wiring connections:
1. A Switchboard listener on `session:state-changed` that calls `contextManager.setSecondaryActive(true)` when state reaches `passive` (Secondary running) and `setSecondaryActive(false)` when state reaches `stopped`
2. A Wire topology subscription on Primary's session that receives DIRECTIVE envelopes from Secondary containing face prompt updates and routes them to `contextManager.receiveSecondaryUpdate(facePrompt)`
3. A face prompt send in the hook handler's existing Wire snapshot path so Mind cycle's composeFacePrompt output reaches Primary

Purpose: Close the single verification gap — SES-02 (Secondary as cognitive authority) and CTX-02 (referential framing in live face prompt) both depend on this pipeline being active at runtime.

Output: Modified reverie.cjs and hook-handlers.cjs with production wiring, updated tests confirming end-to-end data flow.
</objective>
