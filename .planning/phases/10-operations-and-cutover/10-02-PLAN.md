    - run([]) executes 6 stages in order: Docker, Neo4j, Graphiti API, MCP Session, Env Vars, Canary Write/Read
    - When Docker stage fails, stages 2-4 and 6 are SKIP'd (they depend on Docker); stage 5 (Env Vars) still runs
    - When Neo4j fails but Docker passes, stages 3-4 and 6 are SKIP'd; stages 1 and 5 show their real results
    - run(['--pretty']) calls formatHealthReport after collecting results
    - run(['--verbose']) passes { verbose: true } to each stage function
    - Output JSON result: { command: 'health-check', timestamp, stages: [{name, status, detail}], summary: { passed, total, ok } }
    - ok is true only when all non-SKIP stages are OK or WARN
  </behavior>
  <action>
    Create `dynamo/lib/switchboard/health-check.cjs` with module identity block `// Dynamo > Switchboard > health-check.cjs`.

    Implement `async function run(args = [], pretty = false)`:

    1. Parse args: extract `--verbose` flag, extract `--pretty` flag (also check the `pretty` parameter).
    2. Define the 6 health-check stages by index using HEALTH_STAGES: `[stageDocker, stageNeo4j, stageGraphitiApi, stageMcpSession, stageEnvVars, stageCanaryWriteRead]`.
    3. Define dependency map -- which stages to skip when a prior stage fails:
       - Docker FAIL -> skip Neo4j, Graphiti API, MCP Session, Canary (indices 1,2,3,5 within health stages)
       - Neo4j FAIL -> skip Graphiti API, MCP Session, Canary (indices 2,3,5)
       - Graphiti API FAIL -> skip MCP Session, Canary (indices 3,5)
       - Env Vars (index 4) has no downstream dependencies -- always runs
    4. Execute stages sequentially, applying skip logic. For each:
       - If skipReason is set and stage depends on failed stage: `{ status: 'SKIP', detail: '(skipped -- {skipReason} failed)' }`
       - Otherwise: `await stageFunction({ verbose })`. If FAIL, set skipReason.
    5. Build result object: `{ command: 'health-check', timestamp: new Date().toISOString(), stages: [...], summary: { passed, total, ok } }`.
       - `passed` = stages with OK or WARN status
       - `total` = stages with non-SKIP status
       - `ok` = `passed === total`
    6. If pretty: call `formatHealthReport(result)`. Always: call `output(result)`.

    Export: `module.exports = { run }`.

    For the test file `dynamo/tests/health-check.test.cjs`:
    - Mock stages.cjs by creating a wrapper that injects fake stage results
    - Test: when all stages return OK, summary.ok is true and passed equals total
    - Test: when stageDocker returns FAIL, stages 2-4 and 6 are SKIP'd, stage 5 still runs
    - Test: when stageNeo4j returns FAIL but Docker OK, stages 3-4 and 6 SKIP'd

    ```javascript
    const config = loadConfig();
    const resp = await fetchWithTimeout(config.graphiti.health_url, {}, config.timeouts.health);

    ```
