<objective>
Build the Commutator I/O bridge service -- the translation layer between Claude Code hook payloads and Switchboard domain events. Commutator inspects hook payloads, applies tool-aware semantic routing to emit domain-specific events, and supports bidirectional I/O with outbound adapter registration.

Purpose: Commutator is the I/O bridge. It transforms raw Claude Code hook JSON payloads into meaningful domain events that services subscribe to via Switchboard. Without Commutator, every service would need to parse raw hook payloads independently. The outbound adapter pattern enables injecting context back into Claude Code sessions (stdout for hooks now, Wire for inter-session in Phase 6).
Output: `core/services/commutator/commutator.cjs` with full test coverage via `core/services/commutator/__tests__/commutator.test.js`
</objective>

         - `const overrideKey = domain + ':' + hookPayload.hook_event_name;`
         - `const action = TOOL_ACTION_OVERRIDE[overrideKey] || TOOL_ACTION_MAP[hookPayload.hook_event_name] || 'unknown';`
         - Return `domain + ':' + action` (e.g., 'file:changed', 'shell:executed', 'file:pending')
