    Container internals (per D-01, D-02, D-03, D-04, D-05):
    - `_registry` is a `Map<string, RegistryEntry>` where RegistryEntry = `{ factory, instance: null, lifetime: 'singleton'|'factory', deps: string[], tags: string[], aliases: string[], mapDeps: Record<string,string>, config: Record<string,any>, deferred: boolean }`
    - `_aliases` is a `Map<string, string>` mapping alias names to primary binding names
    - `bind(name, factory, options = {})` -- registers factory with metadata. Options: `{ deps, tags, aliases, lifetime, mapDeps, config, deferred }`. Defaults: deps=[], tags=[], aliases=[], lifetime='singleton', mapDeps={}, config={}, deferred=false. Also registers all aliases into `_aliases`.
    - `singleton(name, factory, options)` -- calls bind with lifetime forced to 'singleton'
    - `factory(name, factory, options)` -- calls bind with lifetime forced to 'factory'
    - `resolve(name)` -- looks up `_aliases` first (alias -> primary name). For singleton: if instance is null, call `_instantiate(name)` and cache in entry.instance, then return. For factory: always call `_instantiate(name)` and return fresh instance.
    - `_instantiate(name)` -- gets the entry, calls `entry.factory()` to get the raw service (unwraps Result if factory returns Result), returns the raw instance. Does NOT call init() -- that is lifecycle's job. Container creates, lifecycle initializes.
    - `resolveTagged(tag)` -- iterates `_registry`, collects all entries where `entry.tags.includes(tag)`, returns array of `{ name, entry }` objects.
    - `has(name)` -- checks `_registry.has(name) || _aliases.has(name)`
    - `getMetadata(name)` -- returns `{ deps, tags, aliases, lifetime }` or null
    - `getBootOrder()` -- runs Kahn's algorithm (topological sort) on the dependency graph derived from _registry deps arrays. Returns `Result<string[]>`. On cycle: `err('CYCLE_DETECTED', '...', { cycle: [...] })`.
    - `getRegistry()` -- returns the internal Map for lifecycle/diagnostics access.

    Export: `module.exports = { createContainer }` where createContainer() returns the container object (NOT via createContract -- container is infrastructure, not a service).

    Error codes:
    - `BINDING_NOT_FOUND` when resolve() cannot find name or alias
    - `BINDING_EXISTS` when bind() called with existing name (prevent double registration)
    - `CYCLE_DETECTED` when getBootOrder() finds circular deps
    - `INSTANTIATION_FAILED` when factory() throws or returns Err

    CJS conventions: `'use strict'` at top, JSDoc typedefs, `module.exports = { createContainer }` at bottom. Require only from `../../lib/result.cjs` (ok, err).
