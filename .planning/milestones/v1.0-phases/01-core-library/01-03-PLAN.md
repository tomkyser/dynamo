<objective>
Build the hierarchical configuration loader that reads config from 5 precedence levels (defaults, global config.json, project .dynamo/config.json, DYNAMO_* env vars, runtime options), validates against a schema, and returns typed Result objects. Then wire all lib/ modules together via a barrel export (index.cjs).

Purpose: Every Dynamo component needs validated configuration at boot. The config loader is the last foundational piece -- it depends on result types, schema validation, and path resolution. The barrel export provides the single import point that all downstream code uses.
Output: lib/config.cjs with hierarchical loading, lib/index.cjs barrel, root config.json, comprehensive tests.
</objective>

    **`deepMerge(target, source)`:**
    - Create a new object `result = { ...target }`.
    - For each key in source:
      - If source[key] is a non-null, non-array object AND result[key] is a non-null, non-array object: recurse `result[key] = deepMerge(result[key], source[key])`.
      - Otherwise: `result[key] = source[key]` (scalar, array, or null replacement).
    - Return result. Never mutate inputs.

    **`coerceValue(value)`** (internal helper, not exported):
    - If value === `'true'`, return `true`.
    - If value === `'false'`, return `false`.
    - If value === `'null'`, return `null`.
    - If value matches `/^-?\d+(\.\d+)?$/` (numeric), return `Number(value)`.
    - Otherwise return value as-is (string).

    **`envToConfig(env)`:**
    - `const prefix = 'DYNAMO_';`
    - Iterate `Object.entries(env)`. Skip keys not starting with prefix.
    - For matching keys: strip prefix, lowercase, split on `_` to get path segments.
    - Build nested object by walking path segments.
    - Apply `coerceValue()` to the final value.
    - Return the constructed config object.

    **`loadConfig(options)`:**
    - `options` shape:
      ```
      {
        defaults: Object,           // Level 1: built-in defaults
        schema: Object,             // Schema for validation (passed to validate())
        globalConfigPath: string,   // Level 2: path to global config.json (optional, defaults to discovered root/config.json)
        projectConfigPath: string,  // Level 3: path to project .dynamo/config.json (optional)
        env: Object,                // Level 4: env vars object (optional, defaults to process.env)
        runtimeOverrides: Object,   // Level 5: runtime overrides (optional)
      }
      ```
    - Build merged config by applying deepMerge in precedence order:
      1. Start with `options.defaults || {}`
      2. If `globalConfigPath` exists (check with `fs.existsSync`), read JSON and deepMerge. Use `JSON.parse(require('node:fs').readFileSync(path, 'utf8'))` for synchronous reads (per research open question -- synchronous is better for boot).
      3. If `projectConfigPath` exists, read JSON and deepMerge.
      4. deepMerge with `envToConfig(options.env || process.env)`.
      5. deepMerge with `options.runtimeOverrides || {}`.
    - If `options.schema` is provided, validate merged result with `validate(merged, options.schema)`.
      - If validation returns Err, return that Err (per D-08: fail fast).
      - If validation returns Ok, return `ok(validatedResult)`.
    - If no schema provided, return `ok(merged)`.

    Export: `module.exports = { loadConfig, deepMerge, envToConfig };`

    JSDoc on all three exported functions (per D-11). Include `@typedef` for the options parameter.
