<objective>
Wire Magnet state persistence into Mode Manager, Session Manager, and all three CLI command handlers so that operational state survives process exit and is readable across CLI invocations.

Purpose: Per D-03/D-07/D-08, every mode change and session state transition must write to Magnet immediately, and CLI commands must read from Magnet instead of in-memory state. This is what makes `reverie status` work from a separate terminal window. The start command must do a clean start (per D-03: kill stale processes, clear state, spawn fresh). The stop command must persist shutdown state and clean up.

Output: Modified Mode Manager and Session Manager with Magnet persistence on every mutation, modified CLI handlers reading/writing persisted state.
</objective>
