Update modules/reverie/hooks/hook-handlers.cjs:

The hook handlers currently assume they are called in a context with live services. In the daemon model, they are -- but they need to inspect `payload.env.SESSION_IDENTITY` to know which session type is firing the hook.

Add session identity dispatch to key handlers:

1. `handleSessionStart(payload)`:
   - Extract identity: `const identity = payload.env?.SESSION_IDENTITY || 'primary'`.
   - If `identity === 'secondary'`: call `handleSecondaryStart(payload)` -- Secondary-specific init (load Self Model, begin Wire listening). Return `{}` (Secondary doesn't inject additionalContext on itself).
   - If `identity === 'tertiary'`: call `handleTertiaryStart(payload)` -- Tertiary-specific init (sublimation cycle). Return `{}`.
   - Default (primary): current handleSessionStart logic unchanged (face prompt injection via additionalContext).

2. `handleUserPromptSubmit(payload)`:
   - Extract identity from `payload.env?.SESSION_IDENTITY`.
   - If `identity === 'secondary'`: Secondary's prompt processing (experience evaluation). Different response than primary.
   - If `identity === 'tertiary'`: Tertiary's prompt processing (sublimation). Different response.
   - Default (primary): current logic (formation, recall, context injection).

3. Other handlers: add identity extraction but most can pass through unchanged since they primarily serve the Face session. Add null-guard: `if (!payload.env) payload.env = {}`.

Create helper functions:
- `handleSecondaryStart(payload)` -- placeholder that returns `{}` for now (Secondary init logic already exists in session-manager).
- `handleTertiaryStart(payload)` -- placeholder that returns `{}` for now.
