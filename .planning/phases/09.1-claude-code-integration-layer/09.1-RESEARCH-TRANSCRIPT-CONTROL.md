# Transcript Control Research: claude-mem Endless Mode Analysis

## Summary

Claude Code exposes a `transcript_path` to hook handlers â€” the path to the JSONL file that IS the conversation context window. This file is **read/write**. The claude-mem plugin's "endless mode" beta demonstrates that direct transcript manipulation is viable for active context window management.

## Mechanism Discovery

### Three Techniques Stacked Together

#### 1. `additionalContext` Injection (Official API)
- Hook responses include `hookSpecificOutput.additionalContext` to inject content into context
- Already used in Reverie's Phase 8 hooks for face prompt injection
- Official, documented Claude Code hook contract

#### 2. Direct Transcript File Manipulation (Undocumented)
- Claude Code passes `transcript_path` to hook handlers via stdin JSON
- The transcript is a JSONL file â€” each line is a conversation event (user turn, assistant turn, tool_use, tool_result)
- claude-mem reads this file, finds `tool_use` blocks by `tool_use_id`, and **replaces `block.input` with `{}`**
- This effectively **removes data from the active context window**
- The function `clearToolInputInTranscript()` in `src/hooks/context-injection.ts` implements this

#### 3. Claude Agent SDK Observer Subprocess
- Spawns a separate Claude process via `@anthropic-ai/claude-agent-sdk`
- Observer receives raw tool output, generates ~500 token compressed "observation"
- Observation replaces the full output in context via additionalContext
- Full output stored in SQLite for later retrieval

### Data Flow

```
User turn â†’ Claude calls tool â†’ tool returns 50K of output
  â†“
PostToolUse hook fires (receives transcript_path, tool_use_id, tool_input, tool_response)
  â†“
Hook sends tool output to background worker (HTTP localhost:37777)
  â†“
Worker spawns observer Claude via Agent SDK
  â†“
Observer compresses 50K â†’ ~500 token observation
  â†“
Hook modifies transcript file: replaces 50K tool output with {}
  â†“
Hook returns observation via additionalContext
  â†“
Net result: context grew by ~500 tokens instead of ~50K
```

### Hook Input Contract

Hooks receive via stdin JSON:
```json
{
  "session_id": "claude-session-123",
  "cwd": "/path/to/project",
  "tool_name": "Read",
  "tool_input": { "file_path": "/src/index.ts" },
  "tool_response": "file contents...",
  "transcript_path": "/path/to/transcript.jsonl"
}
```

### Transcript JSONL Structure

Each line is a JSON object representing a conversation event. Tool use blocks contain:
- `tool_use_id` â€” unique identifier for the tool invocation
- `block.input` â€” the tool's input parameters (this is what gets cleared)
- Tool result blocks reference the same `tool_use_id`

### SessionStart Re-injection

The SessionStart hook matcher is `startup|clear|compact` â€” meaning context re-injection happens:
- On fresh session start
- After `/clear`
- After compaction events

This is how memory survives compaction â€” context is re-injected from the database after compaction clears the window.

## Implementation Details from Source

### Key Files (claude-mem beta/endless-mode)
- `src/hooks/save-hook.ts` â€” PostToolUse hook, core endless mode entry point
- `src/hooks/context-injection.ts` â€” Transcript clearing + observation markdown formatting
- `src/hooks/hook-response.ts` â€” Builds Claude Code hook response JSON with additionalContext
- `src/hooks/context-hook.ts` â€” SessionStart context injection
- `src/services/worker/SessionManager.ts` â€” Event-driven session lifecycle, observation wait mechanism
- `src/services/worker/SDKAgent.ts` â€” Spawns Claude subprocess via Agent SDK

### Hook Registration (hooks.json)
| Hook Event | Script | Purpose |
|---|---|---|
| SessionStart | smart-install.js + context-hook.js + user-message-hook.js | Dependency check, inject past observations, display to user |
| UserPromptSubmit | new-hook.js | Create/resume session in worker DB, start SDK agent |
| PostToolUse | save-hook.js | Core endless mode â€” capture, compress, inject, clear |
| Stop | summary-hook.js | Generate session summary |
| SessionEnd | cleanup-hook.js | Mark session complete, cleanup |

### Blocking Behavior
- Standard mode: save-hook is fire-and-forget (~2ms return, async compression)
- Endless mode: save-hook **blocks** up to 110s waiting for observer to generate observation
- Configurable timeout: `CLAUDE_MEM_ENDLESS_WAIT_TIMEOUT_MS` (default 90000ms)

### Skip List
Low-value tools excluded from processing: `ListMcpResourcesTool`, `SlashCommand`, `Skill`, `TodoWrite`, `AskUserQuestion`

## Caveats and Risks

1. **Transcript format is undocumented** â€” JSONL structure could change in future Claude Code versions. No stability guarantee.
2. **Each PostToolUse blocks** for up to 110s in endless mode (observer Claude generation time). Our approach can be faster with structured extraction vs open-ended summarization.
3. **Agent SDK requires separate Claude session** â€” consumes Claude Max quota. Same trade-off as our planned background formation subagents.
4. **Token savings are theoretical** â€” claude-mem's efficiency claims are based on simulations, not production measurements.
5. **Privacy/security** â€” transcript contains full conversation; manipulation must be careful not to corrupt or leak data.

## Implications for Dynamo

### Lithograph Provider (PLT-02)
The transcript file interface maps naturally to a Dynamo provider:
- **Read**: Parse JSONL, query by tool_use_id, by turn type, by time range
- **Write**: Atomic manipulation â€” replace block content, clear inputs, with rollback on failure
- **Query**: Search conversation history for stimulus detection, context analysis
- Provider contract ensures any service or module can consume transcript data safely

### Exciter Service (PLT-03)
The Claude Code integration surface (hooks, agents, settings) needs a single owner:
- Hook registration through a managed contract (not direct settings.json editing)
- Agent definition management (create, update, remove agent .md files)
- Settings.json management at project and user scope
- CLAUDE.md management for project-level instructions
- Modules and extensions go through Exciter to implement Claude Code features

### Context Manager Enhancement
With Lithograph, the Context Manager evolves from passive injection to active sculpting:
- Read conversation history for formation stimulus context
- Replace stale content with Self Model-framed reconstructions
- Proactively manage utilization instead of just monitoring it
- Budget tracker becomes a controller, not just a monitor

### Formation Subagents
Background formation agents can:
- Read transcript via Lithograph for richer stimulus context (not just hook payloads)
- Form fragments from full conversation history, not just the previous turn
- Potentially compress/replace transcript entries with fragment-based summaries

### Compaction Strategy
Instead of desperately checkpointing before PreCompact:
- Proactively trim older turns through Self Model's frame
- Re-inject critical context after compaction via SessionStart hook
- Orchestrate compaction rather than just surviving it
