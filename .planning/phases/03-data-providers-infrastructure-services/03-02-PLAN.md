  <action>
    Create `core/providers/journal/frontmatter.cjs`:
    - `'use strict'`
    - No npm imports -- only built-in string operations

    **parseFrontmatter(content)**:
    - Match frontmatter delimiters: `content.match(/^---\n([\s\S]*?)\n---\n?([\s\S]*)$/)`
    - If no match, return null
    - Extract yaml string and body string
    - Call internal `_parseYaml(yamlStr)` to parse the YAML block
    - Return `{ frontmatter: parsed, body: body.trim() }`

    **_parseYaml(yaml)** internal parser:
    - Split yaml into lines
    - Process line by line, tracking indentation level
    - Use a stack-based approach: each stack frame holds { indent, target, key }
    - For each line:
      1. Calculate indent (number of leading spaces)
      2. Trim the line
      3. Skip empty lines and comments (lines starting with #)
      4. If line matches `- value` (array item): push to current array
      5. If line matches `key: value`: parse value, assign to current object
      6. If line matches `key:` (no value): push new object/array frame based on next line's content
    - Value parsing (_parseValue):
      - `true`/`false` -> boolean
      - `null` or empty -> null
      - Numeric string -> number (use `Number()` with isNaN check)
      - `"quoted"` or `'quoted'` -> strip quotes
      - `[a, b, c]` -> split on `, `, parse each element
      - Otherwise -> string

    **serializeFrontmatter(frontmatter, body)**:
    - Call internal `_serializeYaml(frontmatter, indent=0)` to produce YAML string
    - Return `---\n${yaml}---\n\n${body}`

    **_serializeYaml(obj, indent)** internal serializer:
    - For each key in obj:
      - If value is array: serialize as block array with `- item` syntax
      - If value is object: serialize key, then recurse with indent+2
      - Otherwise: serialize as `key: value` with appropriate quoting

    The parser must handle the Reverie fragment schema which includes nested structures like:
    ```yaml
    type: experiential
    temporal:
      created: 2026-01-01
      decay: 0.5
    associations:
      domains:
        - engineering
        - architecture
      entities:
        - claude
    ```
