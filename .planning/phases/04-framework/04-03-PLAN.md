<objective>
Build the plugin system (FWK-04) and the two-phase lifecycle orchestrator (FWK-03). The plugin system handles manifest loading, validation, dependency checking, and enable/disable. The lifecycle orchestrator runs the register/boot sequence that brings the entire platform up and shuts it down.

Purpose: The lifecycle is the heartbeat of Armature -- it takes a container full of registrations and brings them to life in the correct order. The plugin system is how external code extends the platform. Together they complete the framework's composition layer.

Output: `core/armature/plugin.cjs` with manifest handling and dependency verification. `core/armature/lifecycle.cjs` with register/boot/shutdown orchestration.
</objective>

    validateManifest(manifest):
    - Calls `validate(manifest, PLUGIN_MANIFEST_SCHEMA)` from lib/schema.cjs
    - Returns the validate() result directly

    checkDependencies(manifest, container):
    - Iterates `manifest.dependencies.services` and `manifest.dependencies.providers`
    - For services: checks `container.has('services.' + name)` (prefix with 'services.' because container uses dotted names)
    - For providers: checks `container.has('providers.' + name)` (prefix with 'providers.')
    - Collects missing deps into array
    - If any missing: `err('PLUGIN_MISSING_DEPS', '...', { missing, plugin: manifest.name })`
    - If all found: `ok(undefined)`

    loadPlugin(pluginDir, container):
    1. Read `pluginDir + '/plugin.json'` using `require()` (JSON files are natively require-able in CJS)
    2. Call validateManifest(raw) -- return Err if invalid
    3. Check `manifest.enabled` -- if false, return `err('PLUGIN_DISABLED', '...')`
    4. Call checkDependencies(manifest, container) -- return Err if missing
    5. `try { const entry = require(pluginDir + '/' + manifest.main); }` -- catch errors as `err('PLUGIN_LOAD_FAILED', '...')`
    6. If entry has `register` function: call `entry.register(container)` and return its result
    7. Return `ok({ name: manifest.name, manifest })`

    discoverPlugins(pluginsDir):
    - Uses `require('node:fs').readdirSync(pluginsDir, { withFileTypes: true })`
    - Filters for directories
    - Checks each for existence of `plugin.json` using `require('node:fs').existsSync`
    - Returns array of absolute paths to plugin directories
    - Returns empty array if pluginsDir does not exist (graceful degradation)

    Error codes: PLUGIN_DISABLED, PLUGIN_MISSING_DEPS, PLUGIN_LOAD_FAILED

    Export: `module.exports = { PLUGIN_MANIFEST_SCHEMA, validateManifest, checkDependencies, loadPlugin, discoverPlugins }`

    - createLifecycle(container, options) returns a lifecycle object with register/boot/shutdown methods
    - register() runs registerCore(container) then registerPlugins(container, pluginsDir) per D-11 ordering
    - boot() calls container.getBootOrder() for topological sort, then for each service in order: resolve (create instance), call init(options) with resolved deps and config values, call start(), wrap in facade. Uses sequential for-of with await (not Promise.all) for deterministic ordering.
    - boot() builds the options object for each service by: (a) resolving each dep in mapDeps to its facade instance, (b) merging config values, (c) passing combined object to init()
    - boot() wraps each booted service in a facade via createFacade with the entry's metadata (tags, aliases)
    - boot() stores facades in a Map for later access
    - If a plugin fails to boot, error is logged but core continues (per D-11)
    - shutdown() stops services in reverse topological order, calling stop() on each
    - getFacade(name) returns the facade for a booted service
    - getStatus() returns 'idle' | 'registering' | 'booting' | 'running' | 'shutting-down' | 'stopped'
  </behavior>
  <action>
    Create `core/armature/__tests__/lifecycle.test.js` first (RED). Then implement `core/armature/lifecycle.cjs`.

    createLifecycle(container, options = {}):
    - `_status` starts as 'idle'
    - `_facades` = new Map() for storing booted facades
    - `_bootOrder` = [] for tracking boot sequence (used in reverse for shutdown)
    - `_hookRegistry` = createHookRegistry() from hooks.cjs

    register(registrations):
    - Sets _status = 'registering'
    - `registrations` is an array of { name, factory, options } objects (or the caller pre-registers directly on the container)
    - Alternatively, register() can accept a callback `registerFn(container)` that performs all registrations
    - After core registration, if `options.pluginsDir` provided: calls discoverPlugins() and loadPlugin() for each found plugin
    - Sets _status = 'registered'
    - Returns ok with count of registered bindings

    async boot():
    - Sets _status = 'booting'
    - Gets boot order from `container.getBootOrder()`. If Err (cycle), return that Err immediately.
    - For each name in boot order (sequential for-of with await):
      1. Get registry entry via `container.getRegistry().get(name)`
      2. Call `entry.factory()` to create the raw service instance. If factory returns Result, unwrap it.
      3. Build options object: for each key in entry.mapDeps, resolve the mapped container name and use the facade if available (key in _facades), otherwise use the raw resolved instance. Merge entry.config values.
      4. Call `await Promise.resolve(instance.init(options))` -- normalizes sync/async inits
      5. If init returns Err, and this is a plugin (name starts with 'plugins.'), log warning and continue. If core service, return the Err.
      6. Call `await Promise.resolve(instance.start())` if instance has start method
      7. Create facade: `createFacade(name, instance, { tags: entry.tags, aliases: entry.aliases })`
      8. Store facade in `_facades.set(name, facade)`. Also store under each alias.
      9. Track name in `_bootOrder`
    - After all services booted, wire hooks: if config has hooks section, call `_hookRegistry.loadFromConfig(config)` then `_hookRegistry.wireToSwitchboard(switchboard_facade)`
    - Set _status = 'running'
    - Return ok({ facades: _facades, bootOrder: _bootOrder })

    async shutdown():
    - Sets _status = 'shutting-down'
    - Iterate `_bootOrder` in reverse
    - For each name: get facade from _facades, call `await Promise.resolve(facade.stop())` if stop method exists
    - Set _status = 'stopped'
    - Return ok with count of stopped services

    getFacade(name):
    - Returns _facades.get(name) or _facades.get(alias resolved through container) or null

    getStatus():
    - Returns _status

    Export: `module.exports = { createLifecycle }`

    For tests, create mock services with init/start/stop that record call order:
    - Mock service A (no deps): records init/start/stop calls with timestamps
    - Mock service B (depends on A): verifies A's facade is in init options
    - Mock plugin that fails boot: verify core services still running
    - Verify shutdown reverses boot order
    - Verify async init is properly awaited
