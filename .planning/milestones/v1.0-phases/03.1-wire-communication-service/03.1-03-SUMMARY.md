## Decisions Made
- Transport router selects Channels for urgent/directive messages (low-latency session event push) and relay for background/active messages (bulk data), with fallback in both directions per D-01
- Channels transport meta keys use underscores not hyphens (from_session, urgency_level, message_type, message_id, correlation_id) -- Channels API silently drops hyphenated keys per research findings
- Relay server exports _createServer(config) accepting port:0 for random port assignment -- enables fully isolated parallel test execution
- Relay server does NOT auto-unregister sessions on WebSocket close -- session may reconnect via HTTP or a new WS connection, preserving reconnection resilience per D-10
- Channels transport accepts an injected MCP Server instance (IoC) -- does not create its own. channel-server.cjs in Plan 04 will provide the MCP Server
