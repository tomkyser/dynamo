    **1. Create composite-scorer.cjs** in `modules/reverie/components/recall/`:

    Factory: `createCompositeScorer(options)` with `options.weights` (default SCORING_DEFAULTS) and `options.decayConfig` (default DECAY_DEFAULTS).

    `compositeScore(fragment, queryContext)`:
    - fragment: object with `associations: { domains, entities, attention_tags, self_model_relevance }`, `decay: { current_weight, ... }`, `created` (ISO string)
    - queryContext: `{ activeDomains: [], activeEntities: [], attentionTags: [], referenceTime: Date.now() }`
    - Computes 6 factors:
      1. `domainOverlap`: intersection count / max(fragment domains length, query domains length). Returns 0 if either is empty.
      2. `entityCooccurrence`: intersection count / max(fragment entities length, query entities length). Returns 0 if either is empty.
      3. `attentionTagMatch`: intersection count / max(fragment tags length, query tags length). Returns 0 if either is empty.
      4. `decayWeight`: `Math.min(fragment.decay.current_weight, 1.0)` -- direct from fragment decay state, capped at 1.0.
      5. `selfModelRelevance`: `(smr.identity * 0.3 + smr.relational * 0.5 + smr.conditioning * 0.2)` where smr = fragment.associations.self_model_relevance. Returns 0 if not present.
      6. `temporalProximity`: `Math.exp(-0.1 * daysDiff)` where daysDiff = abs(referenceTime - created) / (86400000). Returns 1.0 for same-day, decays exponentially.
    - Final score: weighted sum of all 6 factors using the weights config.
    - Returns the numeric score (0-1 range due to all factors being 0-1 and weights summing to 1.0).

    `rankFragments(fragments, queryContext, limit)`:
    - Computes compositeScore for each fragment
    - Sorts descending by score
    - Returns top `limit` entries as `[{ fragment, score }]`

    Export: `module.exports = { createCompositeScorer }`

    **2. Create query-builder.cjs** in `modules/reverie/components/recall/`:

    Factory: `createQueryBuilder(options)` with no required dependencies.

    `buildPassiveQuery(stimulus)`:
    - stimulus: `{ domains: [], entities: [], attention_tags: [] }` (extracted from current formation context)
    - Returns Assay-compatible query: `{ criteria: { domains, entities, lifecycle: ['working', 'active'] }, options: { sql: "SELECT fragment_id, fragment_type, current_weight, lifecycle FROM fragment_decay WHERE current_weight > 0.3 AND (lifecycle = 'working' OR lifecycle = 'active') ORDER BY current_weight DESC LIMIT 20" }, limit: 5 }`
    - The SQL retrieves candidate IDs from Ledger; the criteria object is for Journal matching.

    `buildExplicitQuery(conversationContext)`:
    - conversationContext: `{ domains: [], entities: [], attention_tags: [], recentTurns: number }` (broader scope)
    - Returns Assay-compatible query: `{ criteria: { domains, entities, attention_tags, lifecycle: ['working', 'active'] }, options: { sql: "SELECT fragment_id, fragment_type, current_weight, lifecycle FROM fragment_decay WHERE current_weight > 0.1 AND (lifecycle = 'working' OR lifecycle = 'active') ORDER BY current_weight DESC LIMIT 50" }, limit: 15 }`
    - Broader scope: lower decay threshold (0.1 vs 0.3), higher result limit (50 candidates vs 20).

    `extractQueryContext(stimulus, selfModel)`:
    - Helper that produces a `queryContext` object for composite scorer from raw stimulus + Self Model
    - Returns `{ activeDomains, activeEntities, attentionTags, referenceTime: Date.now() }`

    Export: `module.exports = { createQueryBuilder }`

    Both files use `'use strict'`, CJS, JSDoc, options-based DI pattern.
