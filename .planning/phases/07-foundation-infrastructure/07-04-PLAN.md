    **Options (DI):**

    - `options.magnet` - Magnet service instance (for in-memory cache)
    - `options.wire` - Wire service instance (for Ledger writes via write coordinator per D-06)

    - `_versions`: Map tracking current version per aspect, e.g. `{ 'identity-core': 1, 'relational-model': 1, 'conditioning': 1 }`

    **Methods:**

    `async save(aspectName, { frontmatter, body })`:
    1. Validate aspectName is in SM_ASPECTS
    2. Increment version: `_versions[aspectName]++`, set `frontmatter.version = 'sm-{shortname}-v${_versions[aspectName]}'` where shortname maps `identity-core` -> `identity`, `relational-model` -> `relational`, `conditioning` -> `conditioning`
    3. Set `frontmatter.updated = new Date().toISOString()`
    4. Validate frontmatter against the correct schema (identityCoreSchema, relationalModelSchema, or conditioningSchema)
    5. If validation fails, return `err('INVALID_SELF_MODEL', message, { aspect: aspectName, errors })`
    6. Write to Journal: `journal.write(aspectName, { frontmatter, body })` -- file will be `{basePath}/identity-core.md`, etc.
    7. Cache in Magnet: `magnet.set('module', 'reverie', 'self-model.' + aspectName, { frontmatter, body })`
    8. Queue Ledger writes for structured fields via Wire: extract structured data from frontmatter (e.g., `frontmatter.value_orientations` -> write to `sm_value_orientations` table). Use `wire.queueWrite()` with write-intent envelopes for each Ledger table per D-06.
    9. Emit `switchboard.emit('reverie:self-model:updated', { aspect: aspectName, version: frontmatter.version })`
    10. Return `ok({ aspect: aspectName, version: frontmatter.version })`

    `async load(aspectName)`:
    1. Check Magnet cache first: `magnet.get('module', 'reverie', 'self-model.' + aspectName)`
    2. If cached, return `ok(cached)`
    3. Read from Journal: `journal.read(aspectName)` -- parses frontmatter automatically
    4. Validate against schema
    5. If valid, cache in Magnet and return `ok({ frontmatter, body })`
    6. If invalid, return `err('CORRUPTED_SELF_MODEL', ...)`

    `getAspect(aspectName)`: Synchronous Magnet cache read. Returns cached value or null.

    `setAspect(aspectName, data)`: Synchronous Magnet cache write. No persistence (use save for that).

    `getVersion(aspectName)`: Returns current version number for aspect.

    **entropy-engine.cjs**: Create `createEntropyEngine(config)` factory per D-07 and D-08.

    **Config:**
    - `config.sigma` - Standard deviation for gaussian noise (default: 0.05, meaning 5% variance)
    - `config.seed` - Optional seed for deterministic testing (uses `crypto.randomUUID()` if not provided)
    - `config.history` - Array of past session outcomes for conditioned entropy (default: [])

    **Implementation:**
    - Box-Muller transform for gaussian random numbers (zero-dependency):
      ```javascript
      function _gaussianRandom(mean, sigma) {
        const u1 = Math.random();
        const u2 = Math.random();
        const z = Math.sqrt(-2 * Math.log(u1)) * Math.cos(2 * Math.PI * u2);
        return mean + sigma * z;
      }
      ```
    - For seeded mode: use a simple LCG (Linear Congruential Generator) seeded from the config seed string hash, replacing Math.random() calls.

    **Methods:**
    - `applyVariance(traitWeights)`: Takes an object `{ trait1: 0.5, trait2: 0.3, ... }`. Applies gaussian noise (mean=0, sigma=config.sigma) to each value. Clamps results to [0, 1]. Returns new object (does not mutate input).
    - `getState()`: Returns `{ sigma, history, sessionCount }` for persistence.
    - `evolve(sessionOutcome)`: Placeholder for REM integration. Records outcome in history. Adjusts sigma based on outcome quality: if `outcome.quality > 0.7`, reduce sigma by 2% (converging toward good states); if `outcome.quality < 0.3`, increase sigma by 2% (exploring away from bad states). Clamp sigma to [0.01, 0.15].
