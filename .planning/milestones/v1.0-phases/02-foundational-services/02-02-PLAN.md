<objective>
Build the Switchboard event bus service -- the event dispatch backbone of the entire platform. Switchboard wraps node:events EventEmitter with custom prefix-wildcard matching and a dual event type system (actions and filters).

Purpose: Switchboard is the communication primitive. Magnet emits state-change events through it (Plan 03). Commutator routes hook payloads through it (Plan 04). Every future service subscribes to Switchboard events. The wildcard dispatch layer is the primary technical challenge -- EventEmitter does NOT natively support pattern matching.
Output: `core/services/switchboard/switchboard.cjs` with full test coverage via `core/services/switchboard/__tests__/switchboard.test.js`
</objective>

  <behavior>
    - on(eventName, handler, options): registers handler. options.type = 'filter' for filter handlers, default is 'action'. options.priority = number (default 100, lower first). Returns removal function.
    - off(eventName, handler): removes specific handler by reference
    - emit(eventName, payload): dispatches to all matching action handlers (exact + wildcard). Fire-and-forget, returns undefined.
    - filter(eventName, payload): dispatches to all matching filter handlers in priority order. Returns Ok(transformedPayload) or Err if halted.
    - Wildcard matching: pattern 'prefix:*' matches any event starting with 'prefix:'. Only suffix wildcard supported (D-05).
    - matchesPattern(pattern, eventName): if pattern ends with ':*', check eventName.startsWith(pattern.slice(0, -1)). Otherwise exact match.
  </behavior>

             if (result !== undefined && result !== true) current = result; // transformed payload

           return ok(current);
