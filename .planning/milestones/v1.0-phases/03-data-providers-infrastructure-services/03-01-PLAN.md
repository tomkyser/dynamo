    `createLedger()` factory function:
    - Private state: `_started = false`, `_backend = null`, `_switchboard = null`, `_dbPath = null`

    `async init(options)`:
    - Requires `options.dbPath` (return err('INIT_FAILED', 'Ledger init requires dbPath') if missing)
    - `_switchboard = options.switchboard || null`
    - `_dbPath = options.dbPath`
    - Backend selection per D-04/D-05:
      1. If `options.backend === 'sqlite'`, use createSqliteBackend()
      2. If `options.backend === 'duckdb'`, use createDuckDBBackend()
      3. If no backend specified (default), try DuckDB first: call `createDuckDBBackend()`, call `backend.open(dbPath)`. If open returns Err, fall back to createSqliteBackend().
    - Call `_backend.open(dbPath)`. If Err, return it.
    - Return ok(undefined)

    `start()`: set `_started = true`, return ok(undefined)
    `async stop()`: if `_backend`, call `_backend.close()`. Set `_started = false`. Return ok(undefined).
    `healthCheck()`: return ok({ healthy: _started, name: 'ledger' })

    `async read(id)`:
    - Execute: `SELECT id, data, created_at, updated_at FROM records WHERE id = ?` with [id]
    - If result is empty array, return err('NOT_FOUND', `Record not found: ${id}`, { id })
    - Parse data column from JSON string back to object: `JSON.parse(row.data)`
    - Return ok({ id: row.id, data: parsed, created_at: row.created_at, updated_at: row.updated_at })

    `async write(id, data)`:
    - Serialize data to JSON: `JSON.stringify(data)`
    - Execute upsert: For SQLite use `INSERT INTO records (id, data, updated_at) VALUES (?, ?, datetime('now')) ON CONFLICT(id) DO UPDATE SET data = excluded.data, updated_at = excluded.updated_at` with [id, jsonStr].
      For DuckDB use `INSERT OR REPLACE INTO records (id, data, updated_at) VALUES ($1, $2, current_timestamp)` with [id, jsonStr].
      (Alternatively, detect backend type and use appropriate upsert syntax. Store a `_backendType` string during init for this.)
    - Emit 'data:written' via switchboard: `{ provider: 'ledger', id, data }`
    - Return ok(undefined)

    `async query(criteria)` per D-03:
    - If criteria is empty object or null, SELECT all: `SELECT id, data, created_at, updated_at FROM records`
    - Otherwise, build WHERE clause from criteria keys. For each key in criteria:
      - Use `json_extract(data, '$.key') = ?` for SQLite
      - Use `json_extract_string(data, '$.key') = ?` for DuckDB
      - (Use `_backendType` to choose syntax)
    - Collect params array
    - Execute query, parse each row's data from JSON
    - Return ok(arrayOfRecords)
    - Add `LIMIT` support: if `criteria._limit`, append `LIMIT N` to query

    `async del(id)`:
    - First check if record exists: `SELECT id FROM records WHERE id = ?`
    - If not found, return err('NOT_FOUND', `Record not found: ${id}`, { id })
    - Execute: `DELETE FROM records WHERE id = ?` with [id]
    - Emit 'data:deleted' via switchboard: `{ provider: 'ledger', id }`
    - Return ok(undefined)

    Return `validateDataProvider('ledger', { init, start, stop, healthCheck, read, write, query, delete: del })`
