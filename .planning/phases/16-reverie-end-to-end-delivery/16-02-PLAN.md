<objective>
Create a macOS terminal window spawning abstraction and modify the session spawner to open visible Terminal.app windows for Secondary and Tertiary Claude Code sessions instead of invisible piped background processes.

Purpose: Per D-02, the user explicitly stated "the wire strategy is to spin up 3 terminal windows for the triplets." The current session-spawner.cjs spawns sessions with `stdin: 'pipe', stdout: 'pipe', stderr: 'pipe'`, making them invisible. This plan replaces piped stdio with real visible Terminal.app windows using osascript, and uses temp shell scripts per Pitfall 6 to avoid multi-layer escaping.

Output: `terminal-spawn.cjs` abstraction for macOS terminal window spawning, modified `session-spawner.cjs` that calls it, and unit tests.
</objective>

       - If `!command`, return `err('MISSING_PARAM', 'command is required for terminal window spawning')`
       - Build `scriptPath = '/tmp/dynamo-session-' + title.replace(/[^a-zA-Z0-9_-]/g, '_') + '.sh'`

       - Call `execSync('osascript -e ' + JSON.stringify(appleScript))`
       - Return `ok({ scriptPath, title })`
       - Wrap in try/catch: on error return `err('TERMINAL_SPAWN_FAILED', e.message)`
