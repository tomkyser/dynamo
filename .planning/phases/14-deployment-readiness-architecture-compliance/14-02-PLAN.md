From node:util parseArgs with strict: false:
- With strict: false, unknown flags ARE parsed into values (boolean types assumed)
- Example: `parseArgs({ args: ['--dry-run', '--limit', '10'], strict: false, allowPositionals: true })`
  produces `values: { 'dry-run': true, limit: '10' }` (NOT ignored -- that's strict: true behavior)
- CORRECTION from research: Actually with strict:false and no option definition for --dry-run,
  parseArgs treats unknown boolean-style flags as strings. Need to test exact behavior.
  The safest approach: define the custom options in a command-local parseArgs call within each handler.

   - `--confirm` -> `values.confirm = true`
   - `--batch-size 5` -> `values['batch-size'] = '5'`
