Create lib/linotype/parser.cjs with a single exported `parse(filePath)` function:

1. Read file via `Bun.file(filePath).text()` (async). Return value is synchronous after await.
2. Extract JSON frontmatter: detect `---\n` delimiters, parse content between them as JSON (not YAML -- per D-07).
3. Validate required frontmatter fields: `name` (string), `version` (string), `slots` (object). Throw Error with descriptive message on missing/invalid fields: `Linotype parse error (${filePath}): missing required field "name"`.
4. Extract body: everything after the closing `---\n` delimiter.
5. Detect slot references in body via regex `/\{\{([^#/>!{][\w.@]+)\}\}/g` to build a `detected_slots` array.
6. Return `createMatrix({ ...frontmatter fields, body, raw_frontmatter: parsedJSON })`.

Also export a `parseString(content, sourceName)` function that does the same without file I/O (for testing and inline templates). The `sourceName` parameter is used in error messages instead of filePath.

Module pattern: `'use strict'` at top, `module.exports = { parse, parseString }`.

  <action>
Create lib/linotype/engine.cjs with a single exported `resolve(body, context, options)` function:

The `options` parameter: `{ slots: Object, partials: Map, strict: true }`.
- `slots` is the slot definitions from the Matrix (used to check required/optional/defaults).
- `partials` is a Map of name->body for `{{> partial}}` resolution.
- `strict` (default true): throw on missing required slots.

Resolution order (single pass approach is fine, but resolve in this priority):
1. Strip comments: remove `{{! ... }}` blocks (can be multiline: `{{!` to `}}`).
2. Process raw blocks: replace `{{{raw}}}...{{{/raw}}}` with preserved content (escape inner syntax).
3. Process conditionals: `{{#if slot_name}}...{{/if}}` and `{{#if slot_name}}...{{else}}...{{/if}}`. Truthy = non-empty string, non-null, non-false, non-undefined, non-empty-array. Support nesting.
4. Process iteration: `{{#each collection_slot}}...{{/each}}`. Inside the block: `{{.}}` is current item (for string arrays), `{{.fieldname}}` for object property access, `{{@index}}` is 0-based index. The iteration block body is repeated for each item.
5. Process includes: `{{> partial_name}}` replaced with the partial body from options.partials Map. If partial not found and strict=true, throw Error. The included partial body is itself resolved against the same context.
6. Process variables: `{{slot_name}}` replaced with `String(context[slot_name])`. Dot notation `{{slot.sub}}` resolves nested: `context.slot.sub`. If slot is required (per options.slots) and missing from context, throw Error: `Linotype cast error: required slot "slot_name" missing`. If slot is optional and missing, use slot.default or empty string.

Return the resolved string (not a Slug -- the public cast() API in the next plan wraps it).

Implementation notes:
- Use regex-based approach, not a full tokenizer. The syntax is intentionally constrained.
- Handle nested conditionals by processing innermost first (or use a recursive approach).
- ~200-300 lines of CJS. No external dependencies.
