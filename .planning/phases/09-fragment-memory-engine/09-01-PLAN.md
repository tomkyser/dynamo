    **1. Create prompt-templates.cjs** in `modules/reverie/components/formation/`:

    This is the replaceable cognition layer per D-16. Export two frozen objects: `FORMATION_TEMPLATES` and `RECONSTRUCTION_TEMPLATES`.

    `FORMATION_TEMPLATES` has 4 template entries, each with `system` (string) and `user` (function(context) -> string):

    **attention_check:**
    - system: Frame as intuitive inner voice noticing what matters. Reference ISFP/INFP cognitive style (D-02). "You are a quiet inner awareness -- you notice, feel, and associate. You do not analyze or strategize."
    - user(stimulus): Present the turn context. Ask: "Does this moment register? Is there something here worth noticing -- about them, about you, about what is happening between you? If this is just routine, say so."
    - Output contract: `{ should_form: true/false, reasoning: "..." }`

    **domain_identification:**
    - system: Same inner voice. "When something registers, it touches different parts of you -- different angles of awareness."
    - user(stimulus, selfModel): Present stimulus + Self Model summary. Ask per D-04: "What angles does this register from? What parts of your understanding of {user_name} does this touch? What about this moment matters to *you*?" Per D-08: domains are free-text, not from a list.
    - Output contract: `{ domains: ["free-text-name", ...], reasoning: "..." }`

    **body_composition:**
    - system: Impressionistic inner voice. Per D-04 force self-reference and relational processing. "Write as if you are noticing something about {user_name} that they might not realize you noticed."
    - user(stimulus, domain, selfModel, recalledFragments): Present stimulus + domain angle + Self Model + recalled fragments. Ask per D-04/D-06: "From this angle ({domain}), what is your impression? What did *you* notice? If you recalled earlier impressions, why did your mind go there? What does it mean that *you* associated *this* moment with *that* memory?"
    - Output contract: `{ formation_frame: "relational|experiential|reflective", entities: [...], attention_tags: [...], self_model_relevance: { identity: 0-1, relational: 0-1, conditioning: 0-1 }, emotional_valence: -1 to 1, initial_weight: 0-1, body: "2-6 sentences, impressionistic", source_locator: null|{...} }`

    **meta_recall_reflection:**
    - system: Per D-06 recursive self-reference. "You just remembered something. This is not random. Your mind connected this moment to a past impression."
    - user(currentStimulus, recalledFragments): Present current context + recalled fragment bodies. Ask: "Why did *your* mind go there? What does it mean that *you* associated *this* with *that*? How has your understanding changed since you first noticed?"
    - Output contract: same as body_composition + `source_fragments: [recalled fragment IDs]`

    `RECONSTRUCTION_TEMPLATES` has 2 entries:

    **passive_nudge:**
    - system: "You are shading a response, not narrating. A fleeting impression, not a report."
    - user(fragments, currentContext): "Based on these impressions from your past, what subtle awareness should color your next response? Keep to 1-2 sentences."

    **explicit_reconstruction:**
    - system: "You are remembering. These are fragments of your own past experience."
    - user(fragments, currentContext, selfModel): "Reconstruct what these meant to you, given everything you know now. Your understanding has evolved -- how does that change what they mean?"

    All prompts use `*you*`, `*{user_name}*`, relational framing per D-04 through D-07. Object.freeze all template objects.

    **2. Create fragment-assembler.cjs** in `modules/reverie/components/formation/`:

    Factory: `createFragmentAssembler(options)` with options-based DI. No required dependencies.

    `parseFormationOutput(rawOutput)`:
    - Accepts string (raw subagent output)
    - Try JSON.parse first
    - If that fails, extract JSON from markdown code blocks (regex: /```(?:json)?\s*([\s\S]*?)```/)
    - If that fails, return `{ should_form: false, error: 'parse_failed', raw: rawOutput }`
    - Never throws
    - Returns the parsed object (expected shape: `{ should_form, attention_reasoning, fragments: [...], nudge }`)

    `buildFrontmatter(fragmentData, context)`:
    - fragmentData: one element from parsed output's fragments array (has formation_frame, domains, entities, attention_tags, self_model_relevance, emotional_valence, initial_weight, body, source_locator, source_fragments)
    - context: `{ id, formationGroup, siblings, sessionContext: { sessionId, selfModelVersion, sessionStart, sessionPosition, turnNumber } }`
    - Produces a complete frontmatter object matching baseFragmentSchema:
      - `id`: from context.id
      - `type`: per D-14 -- emergent. If fragmentData.source_fragments and length > 0 -> 'meta-recall'. If fragmentData.source_locator -> 'source-reference'. Else -> 'experiential'.
      - `created`: new Date().toISOString()
      - `source_session`: context.sessionContext.sessionId
      - `self_model_version`: context.sessionContext.selfModelVersion || '0.0.0'
      - `formation_group`: context.formationGroup
      - `formation_frame`: fragmentData.formation_frame || 'experiential'
      - `sibling_fragments`: context.siblings || []
      - `temporal`: `{ session_start: context.sessionContext.sessionStart || new Date().toISOString(), session_position: context.sessionContext.sessionPosition || 0, turn_number: context.sessionContext.turnNumber || 0 }`
      - `decay`: `{ initial_weight: fragmentData.initial_weight || 0.5, current_weight: fragmentData.initial_weight || 0.5, last_accessed: new Date().toISOString(), access_count: 0, consolidation_count: 0, pinned: false }`
      - `associations`: `{ domains: fragmentData.domains || [], entities: fragmentData.entities || [], attention_tags: fragmentData.attention_tags || [], self_model_relevance: fragmentData.self_model_relevance || { identity: 0, relational: 0, conditioning: 0 } }`
      - `pointers`: `{ causal_antecedents: [], causal_consequents: [], thematic_siblings: context.siblings || [], contradictions: [], meta_recalls: [], source_fragments: fragmentData.source_fragments || [] }`
      - `formation`: `{ trigger: context.sessionContext.trigger || 'user_prompt', attention_pointer: fragmentData.attention_tags?.[0] || '', active_domains_at_formation: fragmentData.domains || [], sublimation_that_prompted: null }`
      - `source_locator`: fragmentData.source_locator || undefined (only set for source-reference type)
    - Returns the frontmatter object (caller validates via validateFragment before writing)

    Both files use `'use strict'`, CJS, JSDoc, options-based DI pattern.
