<objective>
Fix three deployment infrastructure gaps in Dynamo:

1. **VERSION file stale**: `dynamo/VERSION` contains `0.1.0` but the project is at `1.3.0` (matching CHANGELOG and git tag `v1.3-M2`).

2. **Install chicken-and-egg**: When `dynamo install` runs from the deployed copy (`~/.claude/dynamo/`), `REPO_ROOT` resolves to `~/.claude/dynamo/` (relative to `__dirname`), so install copies the deployed directory onto itself instead of copying from the actual git repo. The `.repo-path` dotfile (already written by `installShim()`) contains the real repo path -- install should read it to resolve the correct source when running from the deployed location.

3. **Sync misses content changes**: `diffTrees()` uses mtime comparison (`src.mtime > dst.mtime`) to decide what to copy. After install self-copies (touching all live files), destination mtimes are newer than source, so sync sees 0 changes even though content differs. The fix: when files have the same size, do a byte-level content comparison (like `detectConflicts` already does) instead of relying solely on mtime.

Purpose: Without these fixes, the deployment pipeline is broken -- install is a no-op, sync misses real changes, and version reporting is wrong.
Output: Fixed VERSION, install.cjs, sync.cjs with corresponding test updates.
</objective>

  <name>Task 2: Fix sync diffTrees to use content comparison</name>
  <files>subsystems/switchboard/sync.cjs, dynamo/tests/switchboard/sync.test.cjs</files>
  <action>
1. **Refactor diffTrees to accept source/destination directory paths**: Currently `diffTrees(srcFiles, dstFiles)` only receives file maps (relPath -> {mtime, size}). To do byte comparison it needs the actual directory paths. Change the signature to `diffTrees(srcFiles, dstFiles, srcDir, dstDir)` where srcDir and dstDir are optional. When both are provided and files have the same size, do a `Buffer.compare` of the actual file contents (same pattern used in `detectConflicts`). If the content differs, include the file in `toCopy`. Keep the mtime check as a fast path: if sizes differ OR src is newer, copy without reading content. The byte comparison only runs when sizes match AND dst is newer-or-equal (the case install creates).

   Specifically, replace this block in diffTrees:
   ```javascript
   if (src.size !== dst.size || src.mtime > dst.mtime) {
     toCopy.push(relPath);
   }
   ```
   With:
   ```javascript
   if (src.size !== dst.size || src.mtime > dst.mtime) {
     toCopy.push(relPath);
   } else if (srcDir && dstDir) {
     // Same size, dst is newer or equal mtime -- check actual content
     try {
       const srcBuf = fs.readFileSync(path.join(srcDir, relPath));
       const dstBuf = fs.readFileSync(path.join(dstDir, relPath));
       if (Buffer.compare(srcBuf, dstBuf) !== 0) {
         toCopy.push(relPath);
       }
     } catch (e) {
       // If read fails, assume different and copy
       toCopy.push(relPath);
     }
   }
   ```
