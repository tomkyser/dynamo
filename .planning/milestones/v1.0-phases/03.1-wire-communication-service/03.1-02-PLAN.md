  <behavior>
    - createWriteCoordinator({ ledger: mockLedger }) returns coordinator object
    - queueWrite(envelope) adds write intent to internal priority queue, returns ok with position
    - queueWrite with non-write-intent type returns err('INVALID_WRITE_INTENT', ...)
    - processNext() takes next write from queue and executes via ledger.write(), returns ok with result
    - processNext() on empty queue returns ok(null)
    - Writes are processed in urgency order: urgent write-intents before directive before active before background
    - Within same urgency, FIFO order preserved
    - start() begins processing loop (drain queue continuously)
    - stop() stops processing loop, returns ok with count of remaining queued writes
    - getQueueDepth() returns current queue depth
    - flush() processes all queued writes immediately, returns ok with count processed
    - Batching: two consecutive write-intents with same payload.table value are batched into single ledger.write() call with combined data array
    - Batching: write-intents with different payload.table values are NOT batched
    - On ledger.write() failure (err result), coordinator emits 'write:failed' event and continues to next write
    - On ledger.write() success, coordinator emits 'write:completed' event with { sessionId: envelope.from, table: payload.table }
  </behavior>
