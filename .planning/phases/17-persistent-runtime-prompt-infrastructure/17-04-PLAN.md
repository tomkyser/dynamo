Create core/daemon-server.cjs with a `createDaemonServer(state)` function:

The `state` parameter is the daemon state object from daemon.cjs `getState()`.

Internal state (module-level for WebSocket handler access):
- `_sessions` = new Map() (sessionId -> { capabilities, registeredAt, lastSeen })
- `_mailboxes` = new Map() (sessionId -> [envelope, ...])
- `_pendingPolls` = new Map() (sessionId -> { resolve, timer })
- `_modules` = new Map() (moduleName -> { state: 'discovered'|'enabled'|'active'|'disabled', instance: null })
- `_server` = null (the Bun.serve instance)

The function creates and returns the Bun.serve instance:

```javascript
_server = Bun.serve({
  port: state.config?.daemon?.port || parseInt(process.env.DYNAMO_PORT) || 9876,
  fetch(req, server) {
    const url = new URL(req.url);
    const pathname = url.pathname;
    // Route dispatch
  },
  websocket: { /* Wire relay WS handlers */ }
});
```

Route dispatch in fetch():

1. `POST /hook` -> Hook dispatch:
   - Parse JSON body: `{ type, payload, env }`.
   - Attempt to resolve exciter from container: `state.container.resolve('exciter')`.
   - If exciter has dispatchHook (Plan 05 adds it): `await exciter.dispatchHook(type, payload, env)`.
   - If no dispatchHook yet: return `_jsonResponse({})` (graceful no-op until Plan 05 wires it).
   - On handler error: return `_jsonResponse({ error: err.message }, 500)`.
   - On success: return `_jsonResponse(result)`.

2. `POST /cli` -> CLI forwarding:
   - Parse JSON body: `{ command, args, flags }`.
   - Resolve pulley from container: `state.container.resolve('pulley')`.
   - If pulley has executeCommand: call it. Otherwise: return `{ output: 'CLI forwarding not implemented', exitCode: 1 }`.
   - Return `_jsonResponse({ output, exitCode })`.

3. `GET /health` -> Health check:
   - Return `_jsonResponse({ status: 'running', pid: process.pid, port: _server.port, uptime_seconds: Math.floor((Date.now() - new Date(state.startedAt).getTime()) / 1000), modules: Array.from(_modules.entries()).map(([name, m]) => ({ name, state: m.state })) })`.

4. `POST /shutdown` -> Graceful shutdown:
   - Log "Shutdown requested via HTTP".
   - Call `process.emit('SIGTERM')` to trigger the signal handler in daemon.cjs.
   - Return `_jsonResponse({ status: 'shutting_down' })`.

5. `POST /module/enable` -> Module enable:
   - Parse body: `{ module: string }`.
   - Look up module in _modules Map. If not found, check container for discovered modules.
   - Set state to 'enabled'. Store in _modules.
   - Return `_jsonResponse({ status: 'enabled', module: moduleName })`.
   - (Full module lifecycle wired in Plan 06.)

6. `POST /module/disable` -> Module disable:
   - Parse body: `{ module: string }`.
   - Set state to 'disabled' in _modules.
   - Return `_jsonResponse({ status: 'disabled', module: moduleName })`.

7. Wire relay routes (merged from relay-server.cjs):
   - `POST /wire/register` -> Register session: parse `{ sessionId, capabilities }`, add to _sessions, create empty mailbox.
   - `POST /wire/unregister` -> Unregister session: remove from _sessions, flush mailbox, cancel pending poll.
   - `POST /wire/send` -> Send envelope: validate via `validateEnvelope`, deliver to target mailbox or resolve pending poll.
   - `POST /wire/send-batch` -> Batch send: `{ envelopes: [] }`, iterate and send each.
   - `GET /wire/poll?sessionId=X&timeout=Y` -> Long-poll: if mailbox has messages return immediately, else park in _pendingPolls with timer.
   - `GET /wire/health` -> Wire health: return session count, mailbox sizes.

8. `GET /ws` -> WebSocket upgrade:
   - `server.upgrade(req, { data: { sessionId: url.searchParams.get('sessionId') } })`.

websocket handler:
- `open(ws)`: register session from ws.data.sessionId.
- `message(ws, msg)`: parse JSON, handle 'register' and 'send' frame types.
- `close(ws)`: unregister session, cancel pending poll.

9. Any unmatched route -> `_jsonResponse({ error: 'Not found' }, 404)`.

Helper functions (module-internal):
- `_parseJsonBody(req)` -> try req.json(), return null on error.
- `_jsonResponse(data, status)` -> new Response(JSON.stringify(data), { status, headers }).
- `_deliverToMailbox(sessionId, envelope)` -> push to mailbox or resolve pending poll.

Export: `module.exports = { createDaemonServer }`.

Also export for testing: `module.exports = { createDaemonServer, _getInternalState: () => ({ sessions: _sessions, mailboxes: _mailboxes, modules: _modules }) }`.
