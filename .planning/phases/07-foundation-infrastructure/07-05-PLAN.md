<objective>
Build the FragmentWriter abstraction that performs atomic dual-provider writes -- Journal file first, then Ledger association index via Wire -- with rollback on failure.

Purpose: FRG-09 addresses Pitfall 4 (split-storage inconsistency) from research. Without atomic dual-provider writes, a fragment could exist in Journal without Ledger index entries (invisible to recall) or have Ledger entries without a Journal file (confabulation). The FragmentWriter is the single write path for all fragment creation, ensuring consistency.
Output: Working FragmentWriter that writes, validates, and rolls back correctly. This is the final plan in Phase 7 -- it composes all prior work (JSON frontmatter from Plan 01, write coordinator from Plan 02, schemas from Plan 03, constants from Plan 01).
</objective>

    **Options (DI):**
    - `options.journal` - Journal provider instance
    - `options.wire` - Wire service instance (for queueWrite)
    - `options.switchboard` - Switchboard for event emission
    - `options.sessionId` - Current session ID (used in envelope.from)

    **Methods:**

    `generateFragmentId()`:
    Generate a fragment ID per D-10: `frag-${YYYY-MM-DD}-${hex8}` where hex8 is 8 random hex chars from `crypto.randomUUID().slice(0, 8)`.

    `async writeFragment(fragment, body)`:
    Per D-11 -- journal-first with Ledger rollback:
    1. Validate `fragment` against zod schema via `validateFragment(fragment)`. If invalid, return `err('INVALID_FRAGMENT', message, { errors })`.
    2. Determine lifecycle directory: `fragment._lifecycle || 'working'` per D-09.
    3. Write to Journal: `journal.write(fragment.id, { frontmatter: fragment, body })`. The Journal provider uses the base path; the lifecycle directory mapping is handled by path construction in the journal basePath configuration. If this fails, return the error immediately (no Ledger write attempted).
    4. Queue Ledger writes via Wire for each association index table:
       - `fragment_decay`: `{ fragment_id: fragment.id, fragment_type: fragment.type, initial_weight: fragment.decay.initial_weight, current_weight: fragment.decay.current_weight, last_accessed: fragment.decay.last_accessed, access_count: fragment.decay.access_count, consolidation_count: fragment.decay.consolidation_count, pinned: fragment.decay.pinned, lifecycle: 'working' }`
       - `fragment_domains`: One row per domain: `{ fragment_id: fragment.id, domain_id: domain, relevance_score: 0.5 }`
       - `fragment_entities`: One row per entity: `{ fragment_id: fragment.id, entity_id: entity, relationship_type: 'mentioned' }`
       - `fragment_attention_tags`: One row per tag: `{ fragment_id: fragment.id, tag_id: tag }`
       - `formation_groups`: `{ id: fragment.formation_group, stimulus_summary: fragment.formation.trigger, fragment_count: 1, surviving_count: 1, source_session: fragment.source_session }`

       For each table write, create a wire write-intent envelope:
       ```javascript
       const envelope = createEnvelope({
         type: MESSAGE_TYPES.WRITE_INTENT,
         from: options.sessionId || 'reverie',
         to: 'ledger',
         payload: { table: tableName, data: [rowData] },
         urgency: URGENCY_LEVELS.ACTIVE,
       });
       const queueResult = wire.queueWrite(envelope.value);
       ```
    5. If ANY queueWrite returns an error result:
       - Delete the Journal file: `journal.delete(fragment.id)`
       - Return `err('FRAGMENT_WRITE_FAILED', 'Ledger write failed, Journal rolled back', { fragmentId: fragment.id, ledgerError })`
    6. Emit `switchboard.emit('reverie:fragment:written', { id: fragment.id, type: fragment.type })`
    7. Return `ok({ id: fragment.id, path: journalPath })`

    `async deleteFragment(id)`:
    Stub for now -- deletes Journal file and queues Ledger deletions. Implementation placeholder:
    1. Delete from Journal: `journal.delete(id)`
    2. Queue Wire writes to mark fragment_decay.lifecycle = 'archive' (soft delete per D-09)
    3. Return `ok({ id })`

    `async updateFragment(id, frontmatterUpdates, body)`:
