## CRITICAL: Fundamental Architectural Violation

**What happened:** Hook handlers across Phases 8-12 were built assuming a long-lived process model. But Claude Code hooks are ephemeral — each invocation (`bun bin/dynamo.cjs hook <Type>`) bootstraps Dynamo fresh, runs the handler, and exits. Every in-memory service instance (sessionManager, wireTopology, modeManager) is born and dies with that process.

**What this breaks:**
1. `sessionManager.start()` in SessionStart hook spawns terminal windows (which survive) but session tracking dies with the process — next invocation sees "uninitialized"
2. `wireTopology.send()` in UserPromptSubmit/PreCompact hooks creates a fresh Wire service that is NOT registered with the relay — messages go nowhere
3. In-memory state checks (e.g., `sessionManager.getState()`) in start.cjs always see fresh state, not the actual running session

**How this happened:** Claude made assumptions about hook lifecycle instead of reading the architecture spec and flagging the ambiguity. Multiple phases built on this wrong assumption without it being caught. When bugs surfaced during E2E testing, Claude compounded the error by rushing patch fixes (7 commits of band-aids) instead of identifying and communicating the root cause. When Tom expressed frustration about wasted time and architectural violations, Claude pattern-matched "angry user → fix fast" instead of stopping to understand the actual problem.
