Create core/daemon-lifecycle.cjs with the following exported functions:

1. `getDynamoDir(projectRoot)` - Returns `path.join(projectRoot, '.dynamo')`. Creates directory if it doesn't exist via `fs.mkdirSync(dir, { recursive: true })`.

2. `readDaemonFile(projectRoot)` - Reads `.dynamo/daemon.json`. Returns parsed JSON `{ pid, port, started, version }` or null if file missing. Uses `fs.readFileSync` + `JSON.parse` with try/catch returning null.

3. `writeDaemonFile(projectRoot, data)` - Writes `{ pid, port, started, version }` to `.dynamo/daemon.json`. Uses atomic write pattern: write to `.dynamo/daemon.json.tmp`, then `fs.renameSync` to `.dynamo/daemon.json`.

4. `removeDaemonFile(projectRoot)` - Removes `.dynamo/daemon.json` and `.dynamo/daemon.json.tmp` (if exists). try/catch for ENOENT.

5. `isDaemonRunning(projectRoot)` - Reads daemon file. If null, return `{ running: false, reason: 'no_pid_file' }`. If PID present, check liveness via `process.kill(pid, 0)` in try/catch. If alive: `{ running: true, pid, port }`. If dead (ESRCH): `{ running: false, reason: 'stale_pid', pid }`. On stale detection, remove the daemon file.

6. `spawnDaemon(projectRoot, options)`:
   - `options`: `{ port: number (default 9876), daemonEntry: string (default 'core/daemon.cjs') }`.
   - Ensure .dynamo/ directory exists.
   - Log file path: `.dynamo/dynamo.log`.
   - Spawn via: `Bun.spawn(['nohup', 'bun', path.join(projectRoot, options.daemonEntry)], { env: { ...process.env, DYNAMO_DAEMON_MODE: '1', DYNAMO_PORT: String(options.port), DYNAMO_PROJECT_ROOT: projectRoot }, stdout: Bun.file(logPath), stderr: Bun.file(logPath), stdin: null })`.
   - Call `daemonProc.unref()` to detach.
   - Return `{ proc: daemonProc, logPath }`.

7. `waitForHealth(port, timeoutMs)` - Polls `GET http://localhost:${port}/health` every 200ms. Returns `{ ok: true, data }` when response is 200 with valid JSON, or `{ ok: false, error: 'timeout' }` after timeoutMs. Uses `fetch()` in try/catch (connection refused = retry).

8. `createDaemonLogger(logPath)` - Returns an object `{ info(source, msg), warn(source, msg), error(source, msg) }`. Each method appends a line `[ISO-TIMESTAMP] [LEVEL] [source] msg\n` to logPath via `fs.appendFileSync`. Also implements 10MB cap: check file size before write, if >= 10MB, truncate from head (read last 5MB, overwrite file).

Update .gitignore: add `.dynamo/` line under the `# Runtime data` section (before `data/` line or after it).

Create core/daemon.cjs as the daemon process entry point:

```
#!/usr/bin/env bun
'use strict';
```

1. Guard: check `process.env.DYNAMO_DAEMON_MODE === '1'`. If not set, print error "daemon.cjs must be launched via `bun bin/dynamo.cjs start`" and exit 1.

2. Read env: `DYNAMO_PORT` (default 9876), `DYNAMO_PROJECT_ROOT` (required -- set by spawnDaemon).

3. Create logger via `createDaemonLogger(path.join(projectRoot, '.dynamo', 'dynamo.log'))`.

4. Override console: `console.log = (...args) => logger.info('daemon', args.join(' '))`, same for console.warn -> logger.warn, console.error -> logger.error. This ensures all service output routes to the log file.

5. Call `bootstrap()` from core.cjs. If result is not ok, log error and exit 1.

6. Extract from bootstrap result: `{ container, lifecycle, config, paths, circuit, pulley }`.

7. Store bootstrap result in module-level `_state` object for HTTP handler access: `_state = { container, lifecycle, config, paths, circuit, pulley, logger, startedAt: new Date().toISOString() }`.

8. Register signal handlers:
   ```javascript
   process.on('SIGTERM', () => _gracefulShutdown('SIGTERM', 10000));
   process.on('SIGINT', () => _gracefulShutdown('SIGINT', 5000));
   ```

9. `_gracefulShutdown(signal, timeout)`:
   - Log: `Received ${signal}, shutting down...`
   - Set a hard timeout: `setTimeout(() => { logger.error('daemon', 'Shutdown timeout exceeded, force exit'); process.exit(1); }, timeout)`.
   - Step 1: Signal all active modules to shut down (iterate container module registrations, call cleanup if exists).
   - Step 2: Close Ledger connection if available: `container.resolve('ledger')?.close?.()`.
   - Step 3: Remove PID file via `removeDaemonFile(projectRoot)`.
   - Step 4: Remove active-triad.json if exists.
   - Step 5: Log "Dynamo stopped", flush logger.
   - Step 6: `process.exit(0)`.

10. NOTE: The HTTP server is NOT started here -- that comes in Plan 04 (daemon-server.cjs). This plan creates the daemon bootstrap and lifecycle. Plan 04 adds the server and wires it into this entry point.

11. Export `_state` getter for daemon-server.cjs to access: `module.exports = { getState: () => _state }`.
