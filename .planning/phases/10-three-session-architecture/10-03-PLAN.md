<objective>
Build the Session Manager state machine and Mode Manager for Active/Passive operational modes. Session Manager orchestrates session lifecycle through Conductor (spawning) and Wire (communication). Mode Manager controls which operational mode is active and handles automatic fallback.

Purpose: This is the central orchestration layer of Phase 10. Session Manager coordinates the three-way responsibility split (D-03): it directs WHAT to spawn (Conductor executes), WHAT topology to enforce (Wire executes), and transitions through the lifecycle state machine (D-02). Mode Manager implements OPS-01 (Active) and OPS-02 (Passive) with the go/no-go fallback gate.

Output: session-manager.cjs (lifecycle state machine), mode-manager.cjs (Active/Passive with fallback)
</objective>

    Create `modules/reverie/components/session/session-manager.cjs` with `'use strict'` header.

    Require: `{ ok, err } from '../../../../lib/index.cjs'`, session-config.cjs constants (SESSION_IDENTITIES, SESSION_STATES, TRANSITIONS, TOPOLOGY_RULES), Wire protocol (MESSAGE_TYPES, URGENCY_LEVELS).

    `createSessionManager({ conductor, wire, selfModel, switchboard, sublimationLoop, config } = {})`:

    Note: `sublimationLoop` is injected so that upgrade() can retrieve the Tertiary system prompt and deliver it via Wire. This is the delivery path for the operational instructions that drive Tertiary's self-prompting loop.

    Internal state:
    - `_state = SESSION_STATES.UNINITIALIZED`
    - `_secondarySessionId = null`
    - `_tertiarySessionId = null`
    - `_config = config` (from createSessionConfig)

    Helper `_transition(targetState)`:
    - Check TRANSITIONS[_state] includes targetState
    - If not, return `err('INVALID_TRANSITION', \`Cannot transition from ${_state} to ${targetState}\`)`
    - Set `_state = targetState`
    - Emit via switchboard: `'session:state-changed', { from: oldState, to: targetState }`
    - Return `ok({ from: oldState, to: targetState })`

    **start()** (async):
    1. `_transition(SESSION_STATES.STARTING)` -- fail if invalid
    2. Generate sessionId for Secondary: `'reverie-secondary-' + Date.now()`
    3. Spawn Secondary via `conductor.spawnSession({ sessionId, identity: SESSION_IDENTITIES.SECONDARY, env: { relayUrl: wire relay URL, SESSION_IDENTITY: 'secondary', MODEL: config.secondary_model } })`
    4. If spawn fails, `_transition(SESSION_STATES.STOPPED)` and return err
    5. `_secondarySessionId = sessionId`
    6. Register in Wire: `wire.register(sessionId, { identity: 'secondary', capabilities: ['send', 'receive', 'write'], writePermissions: ['ledger', 'journal', 'magnet'] })`
    7. `_transition(SESSION_STATES.PASSIVE)` -- now in Passive mode
    8. Return `ok({ state: _state, secondary: _secondarySessionId })`

    **upgrade()** (async):
    1. `_transition(SESSION_STATES.UPGRADING)` -- validates current state is passive
    2. Generate sessionId for Tertiary: `'reverie-tertiary-' + Date.now()`
    3. Spawn Tertiary via `conductor.spawnSession({ sessionId, identity: SESSION_IDENTITIES.TERTIARY, env: { relayUrl: wire relay URL, SESSION_IDENTITY: 'tertiary', MODEL: config.tertiary_model } })`
    4. If spawn fails, `_transition(SESSION_STATES.PASSIVE)` (fallback, not crash) and return err with reason
    5. `_tertiarySessionId = sessionId`
    6. Register Tertiary in Wire: `wire.register(sessionId, { identity: 'tertiary', capabilities: ['send', 'receive'], writePermissions: [] })` -- Tertiary is read-only
    7. **Send sublimation system prompt to Tertiary via Wire context-injection:**
       - Retrieve the system prompt: `const systemPrompt = sublimationLoop.getSystemPrompt()`
       - Create and send a context-injection envelope:
         ```javascript
         wire.send(wire.createEnvelope({
           from: _secondarySessionId,  // Mind (Secondary) is the authority sending instructions
           to: _tertiarySessionId,
           type: MESSAGE_TYPES.CONTEXT_INJECTION,
           urgency: URGENCY_LEVELS.DIRECTIVE,
           payload: {
             role: 'system_prompt',
             content: systemPrompt,
             source: 'sublimation-loop',
           },
         }))
         ```
       - This delivers the operational instructions that drive Tertiary's self-prompting sublimation cycle.
       - If the send fails, log via switchboard but do NOT fail the upgrade -- Tertiary can receive the prompt on reconnect via the Wire buffered message queue.
    8. `_transition(SESSION_STATES.ACTIVE)` -- full three sessions
    9. Return `ok({ state: _state, secondary: _secondarySessionId, tertiary: _tertiarySessionId })`

    **degrade()** (async):
    1. `_transition(SESSION_STATES.DEGRADING)` -- validates current state is active
    2. Stop Tertiary: `conductor.stopSession(_tertiarySessionId)`
    3. Unregister Tertiary: `wire.unregister(_tertiarySessionId)`
    4. `_tertiarySessionId = null`
    5. `_transition(SESSION_STATES.PASSIVE)`
    6. Return `ok({ state: _state })`

    **stop()** (async):
    1. `_transition(SESSION_STATES.SHUTTING_DOWN)`
    2. If _tertiarySessionId: stop Tertiary first, unregister from Wire
    3. If _secondarySessionId: stop Secondary, unregister from Wire
    4. `_transition(SESSION_STATES.STOPPED)`
    5. Return `ok({ state: _state })`

    **getState()**: Returns `{ state: _state, secondary: _secondarySessionId, tertiary: _tertiarySessionId, config: _config }`

    Export: `{ createSessionManager }`

    2. `createModeManager({ sessionManager, conductor, switchboard, config } = {})`:

    Internal state:
    - `_mode = OPERATIONAL_MODES.PASSIVE`
    - `_startedAt = Date.now()`
    - `_modeChanges = 0`
    - `_lastHealthCheck = null`

    Helper `_setMode(newMode, reason)`:
    - `const oldMode = _mode; _mode = newMode; _modeChanges++`
    - Emit via switchboard: `'mode:changed', { from: oldMode, to: newMode, reason }`

    **requestActive()** (async):
    - If already active, return `ok({ mode: _mode, changed: false })`
    - Call `sessionManager.upgrade()`
    - If upgrade succeeds, `_setMode(OPERATIONAL_MODES.ACTIVE, 'user_requested')`
    - If upgrade fails, keep _mode as passive, return `err('UPGRADE_FAILED', reason)`
    - Return `ok({ mode: _mode, changed: true })`

    **requestPassive()** (async):
    - If already passive, return `ok({ mode: _mode, changed: false })`
    - If in active mode, call `sessionManager.degrade()`
    - `_setMode(OPERATIONAL_MODES.PASSIVE, 'user_requested')`
    - Return `ok({ mode: _mode, changed: true })`

    - `_lastHealthCheck = Date.now()`
    - Get session state from sessionManager.getState()
    - If in active mode and tertiarySessionId exists:

      - If not alive: automatic fallback -- call `sessionManager.degrade()`, `_setMode(OPERATIONAL_MODES.PASSIVE, 'tertiary_health_failure')`

    - If in passive mode and secondarySessionId exists:
      - Check `conductor.getSessionHealth(secondarySessionId)`
      - If not alive: emit `'mode:critical', { reason: 'secondary_not_alive' }` via switchboard
    - If in active mode, check Secondary health too
    - Return `ok({ mode: _mode, healthy: boolean, checked_sessions: [...] })`
