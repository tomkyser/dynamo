    **context-manager.cjs modifications:**

    1. Add `receiveSecondaryUpdate(facePrompt)` method to CONTEXT_MANAGER_SHAPE.optional array.
       This method accepts a face prompt string composed by Secondary (Mind) and updates the internal `_currentFacePrompt` cache. When Secondary is running, face prompt composition is Secondary-driven -- Context Manager is the execution layer that caches what Secondary provides.

    2. Implementation of `receiveSecondaryUpdate(facePrompt)`:
       - Validate facePrompt is a non-empty string
       - Set `_currentFacePrompt = facePrompt`
       - Emit via switchboard: `'context:face-prompt-updated', { source: 'secondary', length: facePrompt.length }`
       - Return `ok({ source: 'secondary', length: facePrompt.length })`

    3. Add `_secondaryActive = false` flag to internal state. Set to true when `receiveSecondaryUpdate` is first called.

    4. Modify `compose()` method: if `_secondaryActive` is true, skip local composition and return `_currentFacePrompt` (Secondary is the authority per D-04). If `_secondaryActive` is false, compose locally as before (Passive mode or pre-Secondary startup).

    5. Add `setSecondaryActive(active)` method to CONTEXT_MANAGER_SHAPE.optional. Sets `_secondaryActive` flag. Called by Session Manager when Secondary starts/stops.

    **hook-handlers.cjs modifications:**

    Add `sessionManager`, `wireTopology`, and `modeManager` to the options accepted by `createHookHandlers`. These are optional (null guard pattern, like formationPipeline) for backward compatibility.

    1. **handleSessionStart** modifications:
       - After existing init logic (warm-start, compact check), if sessionManager is available:
         - Call `sessionManager.start()` (async, fire-and-forget -- don't block hook return)
         - This spawns Secondary and enters Passive mode
       - Existing behavior preserved when sessionManager is null

    2. **handleUserPromptSubmit** modifications:
       - After existing face prompt injection, if wireTopology is available and sessionManager.getState().state !== 'stopped':
         - Create a SNAPSHOT envelope: `{ from: 'primary', to: 'secondary', type: MESSAGE_TYPES.SNAPSHOT, urgency: URGENCY_LEVELS.ACTIVE, payload: { userPrompt: payload.user_prompt, turnNumber: payload.turn_number || _turnCount, toolsUsed: payload.tools_used || [] } }`
         - Send via `wireTopology.send(envelope)` (fire-and-forget, non-blocking)
       - The face prompt injection continues to use `contextManager.getInjection()` which now returns Secondary-driven content when available

    3. **handleStop** modifications:
       - Before existing warm-start persist, if sessionManager is available:
         - Call `sessionManager.stop()` (async -- ordered shutdown of Tertiary then Secondary)
       - Existing snapshot + warm-start persist logic unchanged

    4. **handlePreCompact** modifications:
       - After existing checkpoint, if wireTopology available:
         - Send compaction notification to Secondary: `{ type: MESSAGE_TYPES.SNAPSHOT, urgency: URGENCY_LEVELS.URGENT, payload: { event: 'pre_compact' } }`

    5. **handleSubagentStop** modifications:
       - Existing formation subagent processing unchanged (filters by 'reverie-formation')
       - No additional Phase 10 changes needed here -- formation subagents now run under Mind but the output processing path is the same

    All modifications use the null-guard pattern: `if (sessionManager) { ... }` so that tests without Phase 10 components still pass.
