Build the path resolution and module identity system that all Dynamo components use to locate files and directories. The path registry computes absolute paths for every directory in the Dynamo layout from the discovered root.

Purpose: Eliminates hardcoded paths everywhere. Every service, provider, and module asks the path registry for locations instead of constructing paths themselves (per D-04).
Output: lib/paths.cjs with root discovery + path registry, comprehensive tests.
</objective>
