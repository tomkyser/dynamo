  <action>
    Create `core/sdk/pulley/health.cjs`:
    - `'use strict'` header
    - `const { ok, err } = require('../../../lib/result.cjs');`

    - `function aggregateHealth(facades, registry)`:
      - `const results = []; let overall = 'healthy';`
      - Build a set of primary names (non-alias) from registry: `const primaryNames = new Set(registry ? registry.keys() : facades.keys())`.
      - Iterate facades entries. Skip if the name is not in primaryNames (alias entry).
      - For each facade: if `typeof facade.healthCheck !== 'function'`, skip.
      - Try calling `const hcResult = facade.healthCheck()`.
      - If `hcResult && hcResult.ok && hcResult.value`: push `hcResult.value` to results. If `!hcResult.value.healthy`, set `overall = 'degraded'`.
      - If `hcResult && hcResult.ok === false`: push `{ name: name || 'unknown', healthy: false, error: hcResult.error.message }`. Set `overall = 'unhealthy'`.
      - Catch: push `{ name: name || 'unknown', healthy: false, error: e.message }`. Set `overall = 'unhealthy'`.
      - Return `{ overall, services: results, timestamp: new Date().toISOString() }`.

    - `function analyzeDependencyChain(unhealthyNames, registry)`:
      - Takes array of unhealthy service/provider names and the container registry Map.
      - Build reverse dependency map: for each entry in registry, for each dep in entry.deps, map `dep -> [names that depend on it]`.
      - BFS from each unhealthy name through the reverse dep map: collect all transitively impacted names.
      - Return `{ impacted: [...impactedSet], chains: Object.fromEntries(unhealthyNames.map(n => [n, [...directDependants]])) }`.

    - `function formatDiagnostics(healthReport, analysis)`:
      - Build string: `'Dynamo Health Report\n'`
      - `'Overall: ${healthReport.overall.toUpperCase()}\n'`
      - `'Timestamp: ${healthReport.timestamp}\n\n'`
      - `'Services:\n'`
      - For each service in healthReport.services: `'  ${service.name}: ${service.healthy ? "OK" : "FAIL"}${service.error ? " (" + service.error + ")" : ""}${extraFields}\n'`
        where extraFields are any keys besides name/healthy/error formatted as ` [key=value]`.
      - If analysis and analysis.impacted.length > 0: `'\nImpacted by dependency failures:\n'` then for each: `'  ${name}\n'`.
      - Return the full string.

    - `module.exports = { aggregateHealth, analyzeDependencyChain, formatDiagnostics }`.

    Per D-11 (discretion): lifecycle-driven aggregation. Health iterates facades, not a dedicated service. Research recommended this approach -- the lifecycle already holds all facades in a Map.

  <action>
    Create `core/services/forge/versioning.cjs`:
    - `'use strict'` header
    - `const { ok, err } = require('../../../lib/result.cjs');`

    - `function parseVersion(str)`:
      - Strip leading 'v', 'V', 'dev-', 'dev--', 'D.' prefix: `const clean = str.replace(/^[vVdD][\.\-]*/, '')`.
      - Split by '.': `const parts = clean.split('.').map(Number)`.
      - Return `{ major: parts[0] || 0, minor: parts[1] || 0, patch: parts[2] || 0 }`.

    - `function compareVersions(a, b)`:
      - `const va = parseVersion(a); const vb = parseVersion(b);`
      - Compare major, then minor, then patch. Return -1, 0, or 1.

    - `async function getLatestRelease(owner, repo)`:
      - `const url = \`https://api.github.com/repos/${owner}/${repo}/releases/latest\``.
      - Build headers: `{ 'Accept': 'application/vnd.github.v3+json', 'User-Agent': 'dynamo-platform' }`. If `process.env.GH_TOKEN || process.env.GITHUB_TOKEN` exists, add `'Authorization': \`Bearer ${token}\``.
      - Try `const resp = await fetch(url, { headers })`.
      - If `resp.status === 404`: return `err('RELEASE_NOT_FOUND', 'No releases found')`.
      - If `!resp.ok`: return `err('RELEASE_FETCH_FAILED', \`GitHub API returned ${resp.status}\`)`.
      - Parse: `const data = await resp.json()`.
      - Return `ok({ tag: data.tag_name, version: data.tag_name.replace(/^v/, ''), url: data.html_url, name: data.name, prerelease: data.prerelease })`.
      - Catch: return `err('RELEASE_FETCH_FAILED', \`Network error: ${e.message}\`)`.

    - `async function createRelease(owner, repo, tag, options = {})`:
      - Check auth: `const token = process.env.GH_TOKEN || process.env.GITHUB_TOKEN`. If not present, return `err('NO_AUTH_TOKEN', 'GH_TOKEN or GITHUB_TOKEN environment variable required for creating releases')`.
      - `const url = \`https://api.github.com/repos/${owner}/${repo}/releases\``.
      - Build body: `{ tag_name: tag, name: options.name || tag, body: options.body || '', prerelease: options.prerelease || false, draft: options.draft || false }`.
      - POST with auth header.
      - If `!resp.ok`: return `err('RELEASE_CREATE_FAILED', \`GitHub API returned ${resp.status}\`)`.
      - Parse response. Return `ok({ tag: data.tag_name, url: data.html_url, id: data.id })`.
      - Catch: return `err('RELEASE_CREATE_FAILED', \`Network error: ${e.message}\`)`.

    - `function isNewerAvailable(localVersion, remoteVersion)`:
      - Return `compareVersions(localVersion, remoteVersion) < 0`.

    - `module.exports = { parseVersion, compareVersions, getLatestRelease, createRelease, isNewerAvailable }`.

    Per D-12: Forge-based with GitHub REST API. Uses fetch (zero-dependency). Master branch creates vX.Y.Z releases. Dev branch creates dev-X.Y.Z pre-releases. Token from env var only (never config.json).

    Note: Tests should mock fetch() using Bun's mock facilities to avoid real network calls. Use `const { mock } = require('bun:test')` and mock global fetch.
