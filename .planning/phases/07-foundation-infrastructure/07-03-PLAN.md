<objective>
Define the zod validation schemas for all 5 fragment types and Self Model aspects, create the full 12-table association index DDL, and implement the deterministic decay function.

Purpose: These are the data contracts that every subsequent Reverie component references. The schemas define what a valid fragment looks like. The association index defines how fragments relate to domains, entities, and attention tags. The decay function determines fragment survival. Getting these wrong is irreversible per D-12.

    **association-index.cjs**: Create a `createAssociationIndex(options)` factory following options-based DI pattern. Options: `{ connection }` (DuckDB connection object from ledger backend).

    The `init()` method executes DDL for all 12 tables. Use the exact DDL from the research (lines 416-547 of 07-RESEARCH.md):

    Tables (12 total):
    1. `domains` - Core domain taxonomy (id VARCHAR PK, name, description, parent_domain_id, fragment_count INTEGER DEFAULT 0, weight DOUBLE DEFAULT 1.0, narrative_version, archived BOOLEAN DEFAULT false, created_at, updated_at)
    2. `entities` - Named entities (id VARCHAR PK, name, entity_type, description, first_seen, last_seen, occurrence_count INTEGER DEFAULT 1, archived BOOLEAN DEFAULT false, created_at, updated_at)
    3. `associations` - Weighted edges (id VARCHAR PK, source_id, source_type VARCHAR ['entity'|'domain'], target_id, target_type VARCHAR, weight DOUBLE DEFAULT 0.5, co_occurrence_count INTEGER DEFAULT 1, last_reinforced, created_at, updated_at)
    4. `attention_tags` - Experiential descriptors (id VARCHAR PK, tag VARCHAR NOT NULL UNIQUE, occurrence_count INTEGER DEFAULT 1, co_occurrence_data VARCHAR, first_seen, last_seen, created_at)
    5. `formation_groups` - Fragment stimulus groups (id VARCHAR PK, stimulus_summary TEXT, fragment_count INTEGER DEFAULT 0, surviving_count INTEGER DEFAULT 0, source_session VARCHAR NOT NULL, created_at)
    6. `fragment_decay` - Per-fragment decay tracking (fragment_id VARCHAR PK, fragment_type VARCHAR NOT NULL, initial_weight DOUBLE NOT NULL, current_weight DOUBLE NOT NULL, last_accessed, access_count INTEGER DEFAULT 0, consolidation_count INTEGER DEFAULT 0, pinned BOOLEAN DEFAULT false, lifecycle VARCHAR DEFAULT 'working', created_at, updated_at)
    7. `source_locators` - Source reference pointers (id VARCHAR PK, fragment_id VARCHAR NOT NULL, locator_type VARCHAR NOT NULL, path, url, content_hash, last_verified, created_at)
    8. `fragment_domains` - Join: fragment <-> domain (fragment_id + domain_id composite PK, relevance_score DOUBLE DEFAULT 0.5)
    9. `fragment_entities` - Join: fragment <-> entity (fragment_id + entity_id composite PK, relationship_type VARCHAR)
    10. `fragment_attention_tags` - Join: fragment <-> attention_tag (fragment_id + tag_id composite PK)
    11. `entity_domains` - Join: entity <-> domain (entity_id + domain_id composite PK, strength DOUBLE DEFAULT 0.5)
    12. `domain_relationships` - Domain hierarchy (source_domain_id + target_domain_id + relationship_type composite PK, strength DOUBLE DEFAULT 0.5)

    Return object: `{ init, getTableNames }` where `getTableNames()` returns the array of 12 table name strings.

    All column types use INTEGER (not BIGINT) for counts, VARCHAR for IDs, DOUBLE for weights/scores, TIMESTAMP for dates. This avoids Pitfall 4 (BigInt serialization).

    **decay.cjs**: Create `computeDecay(fragment, config)` and `shouldArchive(fragment, config)` functions per spec Section 3.9.

    Formula: `current_weight = initial_weight * relevance_factor * time_decay * access_bonus`
    - `lambda = base_decay_rate / (1 + consolidation_count * consolidation_protection)`
    - `time_decay = Math.exp(-lambda * daysSinceCreation)`
    - `access_bonus = 1 + (Math.log(1 + access_count) * access_weight)`
    - `relevance_factor = (identity * rw.identity) + (relational * rw.relational) + (conditioning * rw.conditioning)`

    `shouldArchive(fragment, config)` returns `true` if computed weight < archive_threshold AND fragment is NOT pinned.

    Import DECAY_DEFAULTS from constants.cjs. Config parameter merges with defaults. Re-export DECAY_DEFAULTS for convenience.

    For association index tests: Use DuckDB `:memory:` database. Create instance, create connection, pass to createAssociationIndex, call init(), verify tables exist via `SELECT table_name FROM information_schema.tables`.
