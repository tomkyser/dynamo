<objective>
Install MCP SDK + zod dependencies and build Wire's transport layer -- the pluggable abstraction that routes messages through Channels API or HTTP relay based on message characteristics, plus the standalone relay server process.

Purpose: Transport abstraction (D-01) enables Wire to route messages through the optimal transport: Channels for low-latency session event push (urgent/directive), relay for bulk data and fallback. The relay server (D-08) runs as a separate Bun process managed by Conductor. This plan delivers the entire network layer Wire needs.

Output: Transport router, two transport implementations, relay server process, MCP SDK + zod installed.
</objective>

    **Step 2: Create transport interface and router** (`core/services/wire/transport.cjs`):
    - `'use strict'`
    - `const { ok, err, createContract } = require('../../../lib/index.cjs');`
    - `const { URGENCY_LEVELS } = require('./protocol.cjs');`
    - `TRANSPORT_SHAPE = { required: ['send', 'connect', 'disconnect', 'healthCheck', 'isConnected'], optional: ['sendBatch', 'type'] }`
    - `validateTransport(impl)` calls `createContract('transport', TRANSPORT_SHAPE, impl)`
    - `createTransportRouter(transports)` factory:
      - `_transports = transports` (array of transport instances, each with a `.type` property: 'channels' or 'relay')
      - `selectTransport(envelope)`: For `envelope.urgency === 'urgent' || envelope.urgency === 'directive'`, find transport with `type === 'channels'` that is connected. If found, return it. Otherwise fall back to relay. For `envelope.urgency === 'background' || envelope.urgency === 'active'`, prefer relay transport. If relay not connected, fall back to channels.
      - `send(envelope)`: call `selectTransport(envelope).send(envelope)`. Return Result.
      - `sendBatch(envelopes)`: group by selected transport, call `sendBatch` on each transport (or iterate `send` if transport lacks `sendBatch`).
      - `connectAll()`: call `connect()` on each transport, return ok with status array.
      - `disconnectAll()`: call `disconnect()` on each transport.
      - `healthCheck()`: return array of transport health statuses.
    - `module.exports = { TRANSPORT_SHAPE, validateTransport, createTransportRouter }`

    **Step 3: Create Channels transport** (`core/services/wire/transports/channels-transport.cjs`):
    - `'use strict'`
    - This transport wraps an MCP Server instance to emit `notifications/claude/channel` events.
    - `createChannelsTransport(options = {})`:
      - `_mcpServer = options.mcpServer` (injected MCP Server instance -- NOT created here, provided by channel-server.cjs in Plan 04)
      - `_connected = false`
      - `type: 'channels'`
      - `connect()`: set `_connected = true`, return ok(). (Actual MCP connection is managed externally by channel-server.)
      - `disconnect()`: set `_connected = false`, return ok().
      - `isConnected()`: return `_connected && _mcpServer !== null`
      - `send(envelope)`: if not connected, return err('TRANSPORT_DISCONNECTED'). Call `_mcpServer.notification({ method: 'notifications/claude/channel', params: { content: JSON.stringify(envelope.payload), meta: { from_session: envelope.from, urgency_level: envelope.urgency, message_type: envelope.type, message_id: envelope.id, correlation_id: envelope.correlationId || '' } } })`. NOTE: meta keys use underscores not hyphens (Channels silently drops hyphenated keys per research). Return ok().
      - `healthCheck()`: return ok({ type: 'channels', connected: _connected })
    - `module.exports = { createChannelsTransport }`

    **Step 4: Create Relay transport** (`core/services/wire/transports/relay-transport.cjs`):
    - `'use strict'`
    - `const { ok, err } = require('../../../../lib/index.cjs');`
    - `createRelayTransport(options = {})`:
      - `_relayUrl = options.relayUrl || 'http://127.0.0.1:9876'`
      - `_sessionId = options.sessionId`
      - `_connected = false`
      - `_pollTimer = null`
      - `_onMessage = options.onMessage || (() => {})` (callback for received messages)
      - `type: 'relay'`
      - `connect()`: POST to `${_relayUrl}/register` with `{ sessionId: _sessionId }`. If 200, set `_connected = true`. Start poll loop. Return ok().
      - `disconnect()`: clear poll timer, POST to `${_relayUrl}/unregister` with sessionId. Set `_connected = false`. Return ok().
      - `isConnected()`: return `_connected`
      - `send(envelope)`: POST to `${_relayUrl}/send` with JSON body of envelope. Return ok/err based on response.
      - `sendBatch(envelopes)`: POST to `${_relayUrl}/send-batch` with array body. Return ok/err.
      - `_poll()`: GET `${_relayUrl}/poll?sessionId=${_sessionId}&timeout=30000`. On response with messages, call `_onMessage` for each. Schedule next poll.
      - `healthCheck()`: GET `${_relayUrl}/health`. Return ok with status.
    - `module.exports = { createRelayTransport }`

    **Create relay server** (`core/services/wire/relay-server.cjs`):
    - `'use strict'`
    - `const { validateEnvelope } = require('./protocol.cjs');`
    - This file is a standalone entry point (`bun run core/services/wire/relay-server.cjs`), not a library.
    - Configuration via env vars: `WIRE_RELAY_PORT` (default 9876), `WIRE_POLL_TIMEOUT_MS` (default 30000)
    - Internal state:
      - `_sessions = new Map()` (Map<sessionId, { registeredAt, lastSeen }>)
      - `_mailboxes = new Map()` (Map<sessionId, envelope[]>)
      - `_pendingPolls = new Map()` (Map<sessionId, { resolve, timer }>)
    - `Bun.serve({ port: RELAY_PORT, fetch(req, server) { ... }, websocket: { ... } })`
    - HTTP routes:
      - `POST /register`: body `{ sessionId }`. Add to _sessions, create empty mailbox. Return 200 `{ ok: true }`.
      - `POST /unregister`: body `{ sessionId }`. Remove from _sessions, _mailboxes, resolve pending polls. Return 200 `{ ok: true }`.
      - `POST /send`: body is message envelope. Validate with `validateEnvelope()`. Route to target session's mailbox. If target has pending poll, resolve it immediately with the message. Return 200 `{ ok: true, delivered: boolean }`.
      - `POST /send-batch`: body is array of envelopes. Validate and route each. Return 200 with results array.
      - `GET /poll?sessionId=X&timeout=30000`: Long-poll. If mailbox has messages, return them immediately. Otherwise, create pending poll promise that resolves when a message arrives or timeout expires. Return 200 `{ messages: [...] }`.
      - `GET /health`: Return 200 `{ status: 'ok', sessions: _sessions.size, uptime: process.uptime() }`.
    - WebSocket upgrade at `GET /ws`:
      - `open(ws)`: wait for registration frame `{ type: 'register', sessionId }`.
      - `message(ws, data)`: handle `{ type: 'send', envelope }` frames. Route envelope to target.
      - `close(ws)`: cleanup WS-to-session mapping.
    - Export `_createServer(config)` for testing (returns server instance with custom port).
    - When run as main: `if (require.main === module)` or check `process.argv`, start server with env config.

    **Create relay server tests** (`core/services/wire/__tests__/relay-server.test.js`):
    - Use `_createServer({ port: 0 })` to get random port for test isolation.
    - Test POST /register: returns 200, session appears in /health
    - Test POST /send: message delivered to registered session's mailbox
    - Test GET /poll: returns messages, long-polls when empty
    - Test GET /poll timeout: resolves with empty array after timeout
    - Test POST /send-batch: batch delivery
    - Test GET /health: returns status, session count, uptime
    - Test POST /unregister: session removed
    - Cleanup: call `server.stop()` in afterEach/afterAll

    Per D-08: Relay runs as separate Bun process managed by Conductor. Wire connects as client.
