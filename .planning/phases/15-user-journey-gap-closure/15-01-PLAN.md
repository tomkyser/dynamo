    **Create `modules/reverie/components/cli/stop.cjs`:**

    Per D-01, D-03. Factory function `createStopHandler(context)` taking `{ modeManager, sessionManager, remConsolidator, contextManager }` from context. Returns `Object.freeze({ handle })`.

    `handle(args, flags)` is async. Mirrors handleStop in hook-handlers.cjs fire-and-forget pattern:

    | Current Mode | Action | Result |
    |-------------|--------|--------|
    | dormant | No-op | `ok({ human: 'Reverie is already dormant (no active sessions)', json: { mode: 'dormant', stopped: false }, raw: ... })` |
    | rem | No-op | `ok({ human: 'Reverie is already consolidating memories (REM mode)\nCheck progress: `bun bin/dynamo.cjs reverie status`', json: { mode: 'rem', stopped: false }, raw: ... })` |
    | active or passive | Execute stop sequence | See below |
    | null modeManager | `err('NOT_INITIALIZED', 'Reverie is not running -- nothing to stop. Check status with `bun bin/dynamo.cjs reverie status`')` |

    Stop sequence (fire-and-forget, returns immediately per Research Pitfall 2):
    1. `await modeManager.requestRem('user_stop_command')`
    2. `await sessionManager.transitionToRem()`
    3. Fire-and-forget: `remConsolidator.handleTier3(sessionContext).then(function() { modeManager.requestDormant(); sessionManager.completeRem(); }).catch(function() { sessionManager.completeRem().catch(function() {}); })`
    4. Fire-and-forget: `contextManager.persistWarmStart().catch(function() {})`
    5. Return immediately: `ok({ human: 'Reverie shutdown initiated\nREM consolidation running in background\nMemories will be preserved before sessions terminate', json: { mode: 'rem', stopping: true, rem_initiated: true }, raw: ... })`

    No `--confirm` gate (stop is graceful, not destructive). No `--skip-rem` flag (D-03).

    Both files use `'use strict'` at top, CJS module.exports, JSDoc comments, `const { ok, err } = require('../../../../lib/result.cjs')`.
